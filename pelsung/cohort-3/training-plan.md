# SELISE x Pelsung — Digital Training Program Plan

**Duration:** 8 weeks (April 7 – May 29, 2026)
**Exhibition/Demo Day:** Week of June 1, 2026 (exact date TBD)
**Trainees:** 45 from Pelsung Cohort 3
**Location:** Gelephu, Bhutan

---

## Program Structure Overview

| Week | Phase | Focus |
|------|-------|-------|
| 1 | Foundation | Corporate skills, digital literacy, team formation |
| 2 | Foundation | SDLC, Agile, tools setup, vertical introduction |
| 3–4 | Core Specialization | Vertical-specific deep training |
| 5–6 | Core Specialization | Advanced skills + cross-vertical collaboration |
| 7 | Capstone Project | Cross-vertical teams build a real product |
| 8 | Polish & Exhibition Prep | Testing, deployment, pitch preparation |
| 9* | Exhibition Week | Demo Day / Showcase (*may overlap with Pelsung graduation) |

---

Suggested split (adjust based on aptitude assessments in Week 1):

| Vertical | Target Size | Focus |
|----------|-------------|-------|

## Week-by-Week Plan

### PHASE 1: FOUNDATIONS (Weeks 1–2)

#### Week 1 — Corporate & Professional Skills + Digital Literacy
*All 45 trainees together*

| Day | Topic |
|-----|-------|
| Mon | Welcome & orientation. SELISE culture, values, expectations. Program overview. |
| Tue | How the digital world works: Internet, DNS, HTTP, browsers, client-server model. |
| Wed | Professional skills: Communication, email etiquette, time management, teamwork. |
| Thu | Tools setup: VS Code, Git/GitHub basics, terminal, environment configuration. |
| Fri | Aptitude assessment + vertical preference survey. Team formation. |

**Deliverable:** Each trainee has a working dev environment and GitHub account.

#### Week 2 — SDLC, Agile & Vertical Orientation
*All 45 trainees together (AM) + vertical breakout sessions (PM)*

| Day | AM (All together) | PM (Vertical breakouts) |
|-----|-------------------|------------------------|
| Mon | Software Development Lifecycle — all 7 stages | Dev: HTML/CSS intro. Design: Design thinking intro. QA: What is quality? BA: What is BA? |
| Tue | Agile & Scrum: sprints, standups, retrospectives | Dev: JavaScript basics. Design: Figma setup. QA: Types of testing. BA: Requirements gathering. |
| Wed | Product thinking: user stories, personas, wireframes | Dev: DOM manipulation. Design: Wireframing exercise. QA: Test case writing. BA: User story writing. |
| Thu | Team roles: how PM, Design, Dev, QA, BA work together | Vertical deep-dive sessions |
| Fri | Mini-project kickoff: all verticals collaborate on a small task | Cross-vertical team exercise |

**Deliverable:** Each trainee understands the full SDLC and their vertical's role within it.

---

### PHASE 2: CORE SPECIALIZATION (Weeks 3–6)

#### Software Development (Weeks 3–6)
*(Detailed curriculum exists in `SoftwareDevelopmentCurriculum.docx` — Phases 2–3)*

- **Week 3:** HTML/CSS, responsive design, JavaScript, debugging, intro to vibe coding
- **Week 4:** AI-assisted development (Cursor, Bolt.new), React fundamentals, hands-on frontend project
- **Week 5:** Backend — Node.js, Express, REST APIs, databases (SQL + NoSQL), auth
- **Week 6:** Full-stack integration, DevOps basics, third-party services, security

#### Software Design (Weeks 3–6)

| Week | Focus | Key Skills | Deliverables |
|------|-------|------------|-------------|
| 3 | UX Fundamentals | User research, personas, journey mapping, information architecture | Persona documents, journey maps |
| 4 | UI Design & Prototyping | Figma deep-dive, component design, design systems, typography, color theory | Interactive Figma prototype |
| 5 | Advanced Prototyping & Handoff | High-fidelity mockups, responsive design, developer handoff specs, accessibility (WCAG) | Complete design spec for a product |
| 6 | Design in Practice | Design critiques, usability testing, iteration, working with developers | Revised designs based on user feedback |

#### Software Testing (Weeks 3–6)

| Week | Focus | Key Skills | Deliverables |
|------|-------|------------|-------------|
| 3 | Manual Testing Foundations | Test planning, test case design, boundary analysis, equivalence partitioning, bug reporting | Test plan + test cases for sample app |
| 4 | Testing in Agile | Sprint-based testing, regression testing, exploratory testing, bug lifecycle, Jira/tracking tools | Bug reports with reproduction steps |
| 5 | Automation Basics | Introduction to test automation, Playwright or Cypress basics, writing first automated tests | Automated test suite for a simple app |
| 6 | API Testing & Performance | Postman for API testing, basic performance testing concepts, security testing awareness | API test collection, test summary report |

#### Business Analysis (Weeks 3–6)

| Week | Focus | Key Skills | Deliverables |
|------|-------|------------|-------------|
| 3 | Requirements Engineering | Stakeholder identification, elicitation techniques (interviews, workshops), documenting requirements (BRD, FRD) | Requirements document for sample project |
| 4 | Modeling & Documentation | Process modeling (BPMN basics), use case diagrams, acceptance criteria, backlog grooming | Use case document, process flow diagrams |
| 5 | Data & Analytics for BA | Basic data literacy, KPIs and metrics, dashboard design concepts, reporting | KPI framework, dashboard wireframe |
| 6 | BA in Agile Teams | Sprint planning support, story mapping, UAT planning, stakeholder communication | UAT plan, sprint-ready backlog |

### PHASE 3: CAPSTONE PROJECT (Weeks 7–8)

#### Week 7 — Build Sprint

Cross-vertical teams (3–4 teams of ~11 people each) work on a **real project relevant to GMC**. Each team has developers, designers, testers, and BAs working together.

**Suggested project themes** (aligned with GMC priorities):
- Citizen services portal for GMC residents
- Tourist/visitor experience app for Gelephu
- Local business directory & marketplace for GMC
- Community wellness and cultural heritage platform
- Smart governance dashboard prototype

| Role | Week 7 Responsibilities |
|------|------------------------|
| **BA** | Requirements gathering, user stories, stakeholder interviews, sprint planning |
| **Design** | Wireframes → high-fidelity mockups → developer handoff |
| **Dev** | Frontend + backend implementation, API integration |
| **QA** | Test planning, continuous testing, bug reporting, automation where possible |

**Daily structure:**
- 9:00 — Standup (all teams)
- 9:30–12:30 — Build sprint
- 12:30–13:30 — Lunch
- 13:30–17:00 — Build sprint
- 17:00–17:30 — End-of-day sync + blockers review

#### Week 8 — Polish, Test & Prepare

| Day | Focus |
|-----|-------|
| Mon | Feature freeze. QA intensive — full regression testing. |
| Tue | Bug fixes, UI polish, accessibility review. |
| Wed | Deployment to staging/demo environment. End-to-end walkthrough. |
| Thu | Pitch preparation: each team crafts a 10-minute presentation (problem → solution → demo → impact). Dry runs with trainers. |
| Fri | Final dress rehearsal. All teams present to trainers. Feedback and final adjustments. |

**Deliverables per team:**
- Working deployed application
- 10-minute presentation deck
- Live demo script
- Project documentation (BA requirements doc, design specs, test report)
