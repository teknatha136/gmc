11 Git Commands
A Normal Day With Git

The 5 commands you'll use 95% of the time

terminal — a typical session
# 1. Get the latest from GitHub
$ git pull

# 2. Make changes in VS Code... then check what changed
$ git status

# 3. Stage the files you want to commit
$ git add index.html style.css

# 4. Save a snapshot with a message
$ git commit -m "Add homepage heading"

# 5. Send it to GitHub
$ git push

VS Code's Git panel does all of this with buttons — but knowing the commands helps when things break.

15 Pull Requests
PR

A Pull Request is a conversation about code

"I made these changes. Please review before we merge."

📝

Describe the change

Title, summary, screenshots if it's UI. Why does this change matter?

💬

Reviewers comment

Your First
Repo & Push

By the end of this exercise, you'll have your own GitHub repo with a README pushed from VS Code

06a
19 Exercise
Follow Along · Raise Hand If Stuck

The 7-step journey

  • 1
    Create a GitHub account at github.com (if you don't have one). Use your real name — this is your public work identity.
  • 2
    Install VS Code from code.visualstudio.com and sign in with GitHub. Sign-in lets VS Code push & pull without typing passwords.
  • 3
    Create a new repo on GitHub — name it hello-<yourname> (e.g. hello-karma), set it Public, and leave it empty (no README yet).
  • 4
    Create a local folder, open it in VS Code, and initialize git. Terminal: mkdir hello-<yourname> && cd hello-<yourname> then git init
  • 5
    Add a README.md — write a few lines about yourself: name, vertical guess, and one hobby.
  • 6
    Stage & commit your README. Terminal: git add README.md then git commit -m "Initial commit"
  • 7
    Link the remote & push to the repo you created in step 3, then verify on GitHub. git remote add origin https://github.com/<user>/hello-<yourname>.git
    git branch -M main · git push -u origin main — then refresh the repo page. 🎉
