06 IP Addresses
IP Addresses

Every Device Has an Address

🏠 Think of a Postal Address

  • Your house has an address so people can find it
  • Your device has an IP address so the internet can find it
  • IP = Internet Protocol
  • Every device on the internet needs one

🔢 What an IP Looks Like

192 . 168 . 1 . 1
  • 4 numbers, each 0–255
  • IPv4: ~4 billion addresses (running out!)
  • IPv6: new format, essentially unlimited

selise.ch has the IP address 185.34.33.33 — that's how your browser really finds it

10 HTTP & HTTPS · 10:15
Protocols

HTTP — The Language of the Web

🔓 HTTP

  • HyperText Transfer Protocol
  • The rules for how browsers and servers talk
  • Data travels in plain text
  • Anyone on the network can read it
  • URL starts with http://

🔒 HTTPS — HTTP + Security

  • S = Secure (uses TLS encryption)
  • Data is encrypted end-to-end
  • Padlock icon in the browser bar
  • Required for passwords, payments, forms
  • URL starts with https://

🛡️ Always check for HTTPS before entering a password or card number. If the padlock is missing — don't type it.

11 Status Codes · 10:25
HTTP Status Codes

The Server's Reply Code

Every HTTP response starts with a 3-digit number

200
OK
Request succeeded. Here's your data.
301
Moved
Page has a new permanent address.
404
Not Found
Page doesn't exist on this server.
403
Forbidden
You don't have permission.
500
Server Error
Something broke on the server.

2xx = success · 3xx = redirect · 4xx = your fault · 5xx = server's fault

12 The Full Journey · 10:30
Step by Step

You Type selise.ch

  • ① Browser checks its cache — has it visited selise.ch recently?
  • ② DNS lookup — asks DNS resolver: "what's the IP for selise.ch?" → gets 185.34.33.33
  • ③ Browser opens a TCP connection to that IP address (port 443 for HTTPS)
  • ④ TLS handshake — browser and server agree on encryption keys
  • ⑤ Browser sends: GET / HTTP/1.1 Host: selise.ch
  • ⑥ Server processes the request, finds the homepage files
  • ⑦ Server responds: 200 OK + sends back HTML
  • ⑧ Browser parses HTML, fetches CSS/JS/images, renders the page ✅

20 REST APIs · 11:45
REST

REST — A Common Language for APIs

Representational State Transfer · predictable, resource-based URLs

Base URL: https://api.pelsung.bt

/trainees All trainees in the system
/trainees/42 Trainee with ID 42 specifically
/trainees/42/classes All classes attended by trainee 42
/classes?phase=foundation Filter classes by phase

The URL tells you what resource you're working with. The HTTP method tells you what to do with it.

21 HTTP Methods · 11:55
HTTP Methods

What to Do With the Resource

Method Action CRUD Example
GET Read / fetch data Read GET /trainees → get all trainees
POST Create new record Create POST /trainees → add new trainee
PUT Update existing record Update PUT /trainees/42 → edit trainee 42
DELETE Remove a record Delete DELETE /trainees/42 → remove trainee

🗝️ CRUD = Create, Read, Update, Delete — every database app does these four things

🗃️ SQL Databases

  • Tables with fixed columns
  • Great for structured, relational data
  • PostgreSQL, MySQL, SQLite
  • Used in most SELISE products

📄 NoSQL Databases

  • Stores JSON documents, not tables
  • Flexible schema — fields can vary
  • MongoDB, Firestore, DynamoDB
  • Great for fast-changing data structures
