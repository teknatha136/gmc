Section 1

HTML — The Structure of a Page

HTML stands for HyperText Markup Language. It's how we tell the browser what's on the page: this is a heading, this is a paragraph, this is a list, this is a link. Every website on the internet uses HTML.

Tags

Anatomy of an HTML Tag

HTML works by wrapping your content in tags. Tags come in pairs — an opening tag and a closing tag — and whatever you put between them becomes that kind of element.

example
<h1>Hello, Gelephu!</h1>
  • Opening tag: <h1> — where the element starts
  • Content: the text or other elements inside
  • Closing tag: </h1> — same tag name with a forward slash, where it ends
  • Together this is called an element. The h1 element above is a big heading that says "Hello, Gelephu!"
Attributes

Tags can have extra info called attributes — written inside the opening tag. Example: <a href="https://google.com"> — href is the attribute name, "https://google.com" is its value. This particular attribute tells the browser where the link goes.

Skeleton

Every HTML Page Has the Same Skeleton

No matter how big or small, every HTML file you write will start with these few lines. Memorise this pattern — you'll type it a hundred times.

index.html
<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello, Gelephu!</h1>
    <p>Welcome to my first web page.</p>
  </body>
</html>
  • <!DOCTYPE html> — tells the browser "this is modern HTML". Always the first line.
  • <html> — the root. Everything else lives inside it.
  • <head> — info for the browser: the page title, links to stylesheets, fonts to load. Nothing here is shown on the page itself.
  • <body> — everything the visitor will actually see.
Vocabulary

Common Tags You'll Use Every Day

There are over 100 HTML tags, but you can build most pages with these ten. Learn these and you're well on your way.

Tag What it does Example use
<h1> … <h6> Headings. h1 is biggest. Section titles
<p> A paragraph of text. Body copy
<a href=""> A clickable link. Navigate between pages
<img src=""> An image. Photos, logos, icons
<ul> + <li> Bulleted list. Skills, features
<ol> + <li> Numbered list. Step-by-step
<div> A generic block-level box. Group content for styling
<span> A generic inline piece. Highlight a word inside text
<button> A clickable button. Forms & actions
<input> A text/email/password field. Forms
Section 2

CSS — Style & Layout

CSS stands for Cascading Style Sheets. While HTML says what is on the page, CSS says how it looks — colours, fonts, spacing, layout, animation.

A CSS Rule

Selector · Property · Value

A CSS rule says "apply these styles to these elements." Every rule has three parts: a selector (which elements), one or more properties (what aspect), and the value for each property.

style.css
/* "all h1 elements" */
h1 {
  color: #0066B2;
  font-size: 2.5rem;
  text-align: center;
}
Three kinds of selector

Element — h1, p, button: targets every tag of that type.
Class — .card: every element with class="card". Starts with a dot.
ID — #hero: the one element with id="hero". Starts with a hash.

The Most Important Concept in CSS

The Box Model

CSS sees every element as a rectangular box. The box has four layers, from inside out: content, padding, border, margin. Understanding these four is the key to controlling any layout.

margin
border
padding
content
  • Content — the actual text, image, or inner elements
  • Padding — space inside the border. Breathing room around the content.
  • Border — the edge of the box. Can be thick, thin, visible, or invisible.
  • Margin — space outside the border. Pushes neighbouring elements away.
Remember

Padding = inside. Use padding when you want more space around the content but still inside the element's background/border.
Margin = outside. Use margin to create space between two elements.

Colours & Typography

Colours, Fonts & Text

These are the properties you'll reach for first when making a page look decent. Even just changing the font and background colour makes a page feel custom.

🎨 Colours — four ways to write them

blue (by name), #0066B2 (hex), rgb(0, 102, 178) (RGB), hsl(205, 100%, 35%) (HSL). Hex is most common. SELISE blue is #0066B2.

✏️ Fonts

font-family sets the typeface, font-size the size, font-weight boldness (400 = normal, 700 = bold). Load free fonts from Google Fonts.

📏 Spacing

padding, margin, and gap take lengths like 16px, 1rem, or percentages. rem scales with the user's font size — more accessible.

🧭 Alignment

text-align: center centres text. justify-content and align-items (on a flex container) centre items both horizontally and vertically.

Layout

Flexbox — Line Things Up

Flexbox is the simplest way to arrange elements in a row or column. Add display: flex to any element and its direct children become flex items — they line up, you control the gaps, and it just works.

A
B
C
style.css
.container {
  display: flex;   /* turn on flexbox */
  gap: 1rem;     /* space between items */
  justify-content: center; /* horizontal align */
  align-items: center; /* vertical align */
}
  • justify-content — horizontal alignment. flex-start, center, flex-end, space-between.
  • align-items — vertical alignment, same values.
  • flex-direction: column — stack vertically instead of a row.
  • gap — space between items. Cleaner than using margins.

The HTML is identical. Only the CSS changed. This is the superpower of separation — you can completely redesign a site without touching any of the content.

Linking the Files

How HTML Finds Your CSS

One line inside <head> tells the browser to load your stylesheet:

index.html
<head>
  <title>My Page</title>
  <link rel="stylesheet" href="style.css">
</head>

Class 06b — Key Takeaways

  • HTML is the structure of a page — tags wrap content to say what each piece is. Every HTML page has the same skeleton: <!DOCTYPE html>, <html>, <head>, <body>.
  • Nine tags cover 80% of pages: headings (h1–h6), p, a, img, ul/ol + li, div, span, button, input.
  • CSS is the style of a page — rules of the form selector { property: value; } change how elements look.
  • Three kinds of selector: element (e.g. p), class (.card), ID (#hero).
  • Every element is a box — content, padding, border, margin. Padding is inside, margin is outside. This is the foundation of every layout.
  • Flexbox is how you line things up — add display: flex to a container, use gap, justify-content, and align-items to control the layout.
  • One <link> tag in the <head> connects your HTML to your CSS. Structure and style stay in separate files, but work together.

✍ Practice at Home

  1. Rebuild your "About Me" page from class on your own laptop, from scratch (no looking at class notes). Two files: index.html and style.css.
  2. Open any website (e.g. selisegroup.com, wikipedia.org). Right-click → Inspect to open the browser's developer tools. Hover over elements in the page — you'll see the HTML on one side and the CSS on the other. Look at what tags and properties real websites use.
  3. Add a second page to your project: projects.html. Use the same CSS file (style.css) so both pages look consistent. Add a link from About Me to Projects using an <a href=""> tag.
  4. Play with flexbox. Wrap three <div> "cards" in a container and use display: flex to lay them out side by side. Try justify-content: space-between, gap: 2rem, and resizing the window.
  5. Explore free resources: developer.mozilla.org (MDN) is the best HTML/CSS reference. flexboxfroggy.com is a fun game that teaches flexbox in 24 levels.
  6. Looking ahead to Class 10: start thinking about what your personal site will look like. What sections will it have? What colours & fonts feel like "you"? What skills do you want to feature?
