Section 01

What is JavaScript?

The programming language that powers the web — and increasingly, everything else too.

💡 JavaScript — The Language of the Web

JavaScript (often abbreviated as JS) is a programming language that was created in 1995 by Brendan Eich at Netscape. It was originally designed to make web pages interactive — things like dropdown menus, image sliders, and form validation. But over the past 30 years, it has grown into one of the most widely used programming languages in the world.

JavaScript is special because it is the only programming language that every web browser understands natively. When you visit a website, your browser downloads HTML (structure), CSS (styling), and JavaScript (behavior). Without JavaScript, web pages would be completely static — no interactions, no animations, no real-time updates.

🌐 In the Browser

  • Every browser has a JS engine built in
  • Chrome uses V8, Firefox uses SpiderMonkey
  • Makes pages interactive and dynamic
  • Handles user clicks, form submissions, animations

🖥️ On Servers

  • Node.js lets JS run outside the browser
  • Build backend APIs and web servers
  • Handle database operations
  • Used by Netflix, PayPal, LinkedIn

📱 Everywhere Else

  • Mobile apps (React Native)
  • Desktop apps (Electron — VS Code, Slack)
  • IoT devices and embedded systems
  • AI tool interfaces and chatbots
🎯 Why this matters for you: As a developer in this program, JavaScript will be your primary language. You'll use it to build web interfaces (frontend) and potentially server-side logic (backend). Learning JS well means you can build almost anything.
Section 02

What is Node.js?

JavaScript was trapped in the browser. Node.js set it free.

🟢 Node.js — JavaScript Runtime

For the first 14 years of JavaScript's life, it could only run inside a web browser. If you wanted to build a server, process files, or interact with a database, you had to use a different language like PHP, Java, Python, or Ruby. This meant web developers had to learn at least two languages — one for the frontend and one for the backend.

In 2009, Ryan Dahl created Node.js. He took Chrome's V8 JavaScript engine (the part that actually executes JS code) and made it work outside the browser. Suddenly, JavaScript could read files, start web servers, connect to databases, and do everything other "backend" languages could do.

🔧 What Node.js Is

  • A runtime environment — a place where JS code can execute
  • Built on Chrome's V8 engine (same engine, different environment)
  • Gives JavaScript access to the file system, network, and OS
  • Free, open source, runs on Windows, Mac, and Linux

🚫 What Node.js Is NOT

  • Not a programming language (JS is the language, Node is the environment)
  • Not a framework (Express, Fastify are frameworks that run on Node)
  • Not only for servers (it runs dev tools, scripts, CLI apps too)
  • Not a replacement for browser JS (they coexist)

Why does this matter? Almost every modern web development tool runs on Node.js. When you use VS Code extensions, run a React development server, or install packages — Node.js is doing the work behind the scenes. Even if you never write backend code, you'll use Node.js every single day as a frontend developer.

💡 Simple analogy: Think of JavaScript as a language you speak, and Node.js as a new country where that language is understood. Before Node, JS could only "travel" inside browsers. Now it can go anywhere.
Section 03

npm — Node Package Manager

Why write everything from scratch when millions of packages are free to use?

📦 npm — Your Developer Toolbox

npm (Node Package Manager) comes bundled with Node.js — when you install Node, you automatically get npm. It serves two purposes: it's an online registry (a massive library of free, reusable code packages) and a command-line tool (what you use to install and manage those packages).

The npm registry has over 2 million packages — everything from tiny helper functions to entire frameworks like React. Instead of writing a date formatting library from scratch, you run npm install date-fns and it's ready to use in seconds.

📋 package.json

  • The "ID card" of every Node.js project
  • Lists project name, version, author
  • Lists all dependencies (packages your project needs)
  • Defines scripts (commands you can run)
  • Created with npm init

📁 node_modules/

  • The folder where npm puts downloaded packages
  • Can get very large (hundreds of MB)
  • Never edit files inside it manually
  • Never commit it to git (use .gitignore)
  • Regenerated by running npm install

🔒 package-lock.json

  • Auto-generated file — don't edit it
  • Locks exact versions of every dependency
  • Ensures your teammate gets the same versions
  • Always commit this file to git

Essential npm Commands

Terminal — npm commands
# Create a new project (generates package.json)
npm init
npm init -y     # skip the questions, use defaults

# Install all dependencies listed in package.json
npm install

# Install a specific package
npm install react
npm install -D vite   # install as dev dependency

# Run scripts defined in package.json
npm run dev     # start development server
npm run build   # build for production
npm start       # run the "start" script
🎯 The key commands you'll use daily: npm install (set up a project), npm run dev (start coding), and npm run build (prepare for deployment). That covers 90% of your npm usage.
Section 04

Essential Terminal Commands

The terminal is not scary — it's just a different way to talk to your computer. And it's faster.

⌨️ Terminal — Your Developer Superpower

The terminal (also called command line, console, or shell) lets you control your computer by typing commands instead of clicking around with a mouse. Every professional developer uses the terminal daily — it's essential for running code, managing files, using git, and working with npm.

It might feel unfamiliar at first, but once you learn a handful of commands, you'll find it's often faster than clicking through folders. And some things (like running a server or installing packages) can only be done from the terminal.

Navigation Commands

CommandWhat It DoesExample
pwdPrint working directory — shows where you are/Users/karma/projects
lsList files and folders in current directoryShows all items in the folder
cd folderChange directory — move into a foldercd my-project
cd ..Go up one level (parent directory)From /projects/app → /projects

File & Folder Commands

CommandWhat It DoesExample
mkdir nameCreate a new foldermkdir my-project
touch fileCreate a new empty filetouch index.html
rm fileDelete a file (no undo!)rm old-notes.txt
rm -r folderDelete a folder and everything inside itrm -r temp-folder
clearClear the terminal screenCleans up visual clutter

Developer Workflow Commands

CommandWhat It Does
node file.jsRun a JavaScript file with Node.js
npm run devStart the development server
npm installInstall all project dependencies
git statusSee what files have changed (version control)
code .Open the current folder in VS Code
Ctrl + CStop a running process (e.g., dev server)
⚠️ Be careful with rm: Deleting files from the terminal is permanent — there is no Recycle Bin or Trash. Always double-check what you're deleting. When in doubt, move files to a temporary folder first instead of deleting them.

Putting It All Together — A Typical Workflow

Terminal
# 1. Create a new project folder
mkdir my-first-app
cd my-first-app

# 2. Initialize a Node.js project
npm init -y

# 3. Open in VS Code
code .

# 4. Install dependencies
npm install vite

# 5. Start the development server
npm run dev

# 6. When done, stop the server
# Press Ctrl + C
