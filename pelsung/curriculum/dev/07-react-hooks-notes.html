Section 01 · Foundations

What is a hook?

A hook is a function that adds a feature to your component.

🪝 The 30-second answer

A normal function returns a value. A React component is a function that returns JSX. A hook is a special function that lets your component remember things (like state) and do things (like fetch data) — features it couldn't do on its own.

You've already met one: useState from Class 06. Today we add useEffect and look at how to write your own.

💡
Spot a hook by its name. Built-in or custom, hook names start with use — useState, useEffect, useCounter. The naming isn't optional; React's tools rely on it.
Section 02 · The rule

The one rule of hooks

Always at the top of the component. No exceptions.

📐 Top-level only

Never call hooks inside if, loops, or other functions. Always call them at the top of your component, in the same order, every render.

rule.tsx
// 🛑 Wrong — inside an if function Bad() { if (condition) { const [x, setX] = useState(0); } } // ✅ Right — at the top, branch INSIDE if needed function Good() { const [x, setX] = useState(0); if (condition) { … } }
🔧
The Vite React-TS template ships with the react-hooks ESLint plugin — your editor will warn you in real time if you break this rule. Trust it.
Section 03 · useState

useState — common shapes

A quick refresher and a few patterns you'll use all the time.

🔁 The shape

useState takes the initial value and returns two things: the current value and a setter function. Calling the setter updates the value and re-renders the component.

basic.tsx
const [count, setCount] = useState(0); // ↑ value ↑ setter ↑ initial

🎨 Common shapes

examples.tsx
// number — counters, prices const [count, setCount] = useState(0); // string — input values, names const [name, setName] = useState(""); // boolean — toggles, modal open/close const [isOpen, setOpen] = useState(false); // array of objects — lists of things const [events, setEvents] = useState<GmcEvent[]>([]);

TypeScript usually figures out the type from your initial value. Add the explicit <…> only when you start with an empty array of a specific type.

🧠 useState with an input

A typical pattern: hold the input's text in state, update on every keystroke, render the value back somewhere else.

name-box.tsx
function NameBox() { const [name, setName] = useState(""); return ( <div> <input value={name} onChange={e => setName(e.target.value)} /> <p>Hello, {name || "friend"}!</p> </div> ); }

📋 Updating arrays — one rule

Don't change the array — make a new one. Use the spread operator [...] to add, and .filter() to remove.

arrays.tsx
// 🛑 Won't update the screen events.push(newEvent); setEvents(events); // ✅ Make a new array setEvents([...events, newEvent]); // add setEvents(events.filter(e => e.id !== id)); // remove
💡
Why? React only re-renders when the value actually changes. events.push(...) changes the array in place, so React sees "same array as before, nothing to do." Spreading creates a brand-new array — React notices the change and re-renders.

Section takeaways

  • useState returns [value, setter]. Calling the setter re-renders.
  • Use it for numbers, strings, booleans, and arrays of objects.
  • For arrays — never push, never sort in place. Always make a new array.
  • One rule: hooks at the top of the component, never inside if.
Section 04 · Hands-on 1

Hands-on 1 — Counter (30 min)

A small "GMC events scheduled" counter with three buttons.

Goal Build an EventsCounter

One number on screen. Three buttons: + Add, − Remove (don't go below 0), Reset. Use useState only.

  1. In src/, create a new file events-counter.tsx.
  2. At the top, import the hook: import { useState } from "react";
  3. Make a function: export function EventsCounter() { … }
  4. Inside, add state: const [count, setCount] = useState(0);
  5. Return a <div> showing the count: <p>Scheduled: {count}</p>
  6. Add an Add button: <button className="rounded-md bg-blue-500 px-3 py-1.5 text-white" onClick={() => setCount(count + 1)}>+ Add</button>
  7. Add a Remove button: <button className="rounded-md border border-gray-300 px-3 py-1.5" onClick={() => setCount(count > 0 ? count - 1 : 0)}>− Remove</button>
  8. Add a Reset button: <button className="rounded-md border border-gray-300 px-3 py-1.5" onClick={() => setCount(0)}>Reset</button>
  9. In app.tsx: import { EventsCounter } from "./events-counter"; and render <EventsCounter />.
  10. Run npm run dev. Click each button. Try going below 0 — it should stop at 0.

📄 The final code

events-counter.tsx
import { useState } from "react"; export function EventsCounter() { const [count, setCount] = useState(0); return ( <div className="flex flex-col gap-3 p-4"> <p className="text-lg">Scheduled: {count}</p> <div className="flex gap-2"> <button className="rounded-md bg-blue-500 px-3 py-1.5 text-white" onClick={() => setCount(count + 1)}>+ Add</button> <button className="rounded-md border border-gray-300 px-3 py-1.5" onClick={() => setCount(count > 0 ? count - 1 : 0)}>− Remove</button> <button className="rounded-md border border-gray-300 px-3 py-1.5" onClick={() => setCount(0)}>Reset</button> </div> </div> ); }

Section 05 · useEffect

useEffect — run code after render

Mostly used to load data from the internet.

⚡ The idea

Rendering should be just "describe the screen". But sometimes you need to do something — fetch data, save to localStorage, set up a timer. useEffect says: "after React draws the screen, also run this code."

📐 The shape

shape.tsx
useEffect(() => { // the code that runs AFTER the render }, [/* the list of things to watch */]);

Two pieces — a function (what to do), and an array (when to do it).

📋 Two simple patterns

ArrayEffect runs…Use for
[]Once, when the component first appearsLoading data on first show
[id]Once, plus every time id changesReloading when something changes

Today we'll use [] — fetch data once when the page opens. That alone unlocks a lot.

Section 06 · Fetching

Fetching from an API

The most common use of useEffect.

🌐 The pattern

Use fetch inside the effect, parse the JSON, and put the data into state with the setter. Add a loading flag to show a "Loading…" message while waiting.

posts.tsx
import { useEffect, useState } from "react"; type Post = Readonly<{ id: number; title: string; body: string }>; export function Posts() { const [posts, setPosts] = useState<Post[]>([]); const [loading, setLoading] = useState(true); useEffect(() => { fetch("https://jsonplaceholder.typicode.com/posts") .then(res => res.json()) .then(data => { setPosts(data); setLoading(false); }); }, []); if (loading) return <p className="text-gray-500">Loading…</p>; return ( <ul className="space-y-2 p-4"> {posts.map(p => ( <li key={p.id} className="rounded-md border border-gray-200 bg-white p-3"> {p.title} </li> ))} </ul> ); }

fetch sends an HTTP request to the URL. res.json() parses the response body as JSON. setPosts(data) stores it in state. The component re-renders with the new posts.

⚠️
Empty array [] matters. Without it, the effect would run after every render — fetching forever in a loop. The empty array means "run me only once, on mount."
Section 07 · Hands-on 2

Hands-on 2 — Posts list (60 min)

Load real data from JSONPlaceholder, show a loading state, render the list.

Goal Build a PostsList component

It fetches from https://jsonplaceholder.typicode.com/posts, shows "Loading…" while waiting, then renders the post titles in a list.

  1. Create a new file: src/posts-list.tsx.
  2. Import both hooks: import { useEffect, useState } from "react";
  3. Define the type: type Post = Readonly<{ id: number; title: string; body: string }>;
  4. Create the function shell: export function PostsList() { … }
  5. Inside, add posts state: const [posts, setPosts] = useState<Post[]>([]);
  6. Add loading state: const [loading, setLoading] = useState(true);
  7. Add a useEffect with an empty array: useEffect(() => { … }, []);
  8. Inside the effect, call fetch("https://jsonplaceholder.typicode.com/posts"), then .then(res => res.json()), then .then(data => { setPosts(data); setLoading(false); }).
  9. Below the effect: if (loading) return <p>Loading…</p>;
  10. Then return a <ul className="space-y-2 p-4"> mapping over posts: {posts.map(p => <li key={p.id} className="rounded-md border border-gray-200 bg-white p-3">{p.title}</li>)}
  11. In app.tsx, import and use it: <PostsList />.
  12. Run the dev server. You should see "Loading…" briefly, then a list of post titles.

📄 The final code

posts-list.tsx
import { useEffect, useState } from "react"; type Post = Readonly<{ id: number; title: string; body: string }>; export function PostsList() { const [posts, setPosts] = useState<Post[]>([]); const [loading, setLoading] = useState(true); useEffect(() => { fetch("https://jsonplaceholder.typicode.com/posts") .then(res => res.json()) .then(data => { setPosts(data); setLoading(false); }); }, []); if (loading) return <p className="text-gray-500">Loading…</p>; return ( <ul className="space-y-2 p-4"> {posts.map(p => ( <li key={p.id} className="rounded-md border border-gray-200 bg-white p-3"> {p.title} </li> ))} </ul> ); }

Section 09 · Custom hooks

Custom hooks (a peek)

A function that uses other hooks, named with a use prefix.

🛠️ The idea

If two components share the same useState + useEffect logic, you can pull that logic into one function — a custom hook. Both components call it. No more duplication.

Example — useCounter

use-counter.ts
import { useState } from "react"; export function useCounter(start: number = 0) { const [count, setCount] = useState(start); const add = () => setCount(count + 1); const remove = () => setCount(count > 0 ? count - 1 : 0); const reset = () => setCount(start); return { count, add, remove, reset }; }

Using it

The component shrinks — the logic lives once, in use-counter.ts.

events-counter.tsx — refactored
import { useCounter } from "./use-counter"; export function EventsCounter() { const { count, add, remove, reset } = useCounter(0); return ( <div className="flex flex-col gap-3 p-4"> <p className="text-lg">Scheduled: {count}</p> <div className="flex gap-2"> <button className="rounded-md bg-blue-500 px-3 py-1.5 text-white" onClick={add}>+ Add</button> <button className="rounded-md border border-gray-300 px-3 py-1.5" onClick={remove}>− Remove</button> <button className="rounded-md border border-gray-300 px-3 py-1.5" onClick={reset}>Reset</button> </div> </div> ); }
📁
One rule: the function name must start with use. File name uses kebab-case: use-counter.ts. The function stays camelCase: useCounter.

Section takeaways

  • A custom hook is just a function that uses other hooks.
  • Name it starting with use.
  • Use it to remove duplicated state logic between components.
  • You'll write more of these as you go — for now, this is just a peek.

Recap

What you should be able to do after this class

🔁 useState

Hold numbers, strings, booleans, arrays. Update with the setter. Remember to make a new array — never push.

⚡ useEffect

Run code after render. Use [] to run once on mount. Most common job: fetch data.

🌐 fetch

Call any free API, parse JSON with .json(), store the result in state, render it.

🛠️ Custom hooks

Bundle useState+useEffect logic into a useFoo function. One file, many components reuse it.
