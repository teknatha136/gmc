Section 01

Variables — var, let, const

Variables are named containers for storing data. JavaScript gives you three keywords to create them.

📦 Declaring Variables

A variable is like a labeled box where you store a value. You give it a name, put something inside, and later refer to it by name. JavaScript has three keywords for creating variables: var, let, and const.

variables.js
// var — the old way (avoid in modern JS)
var name = "Karma";

// let — use when the value will change
let age = 25;
age = 26; // ✅ works — let allows reassignment

// const — use by default (value cannot be reassigned)
const city = "Gelephu";
city = "Thimphu"; // ❌ TypeError — const prevents this

Comparison: var vs let vs const

Featurevarletconst
Can be reassigned? Yes Yes No
Can be re-declared in same scope? Yes (dangerous!) No No
Scope Function scope Block scope Block scope
Hoisted? Yes (as undefined) Yes (TDZ — error if used before declaration) Yes (TDZ)
Recommended? ❌ Never ✅ When value changes ✅ Default choice
🎯 Rule of thumb: Always start with const. If you find you need to reassign the value later, change it to let. Never use var — it has confusing scoping behavior that leads to bugs.

Important nuance with const and objects: When you use const with an object or array, the reference is constant (you can't reassign the variable to a different object), but you can still modify the properties inside it.

const with objects
const user = { name: "Karma", age: 25 };

user.age = 26;     // ✅ works — modifying a property
user.city = "Paro"; // ✅ works — adding a property

user = {};         // ❌ error — can't reassign the variable itself
Section 02

Hoisting & Scoping

Understanding where variables live and how JavaScript processes your code.

🏗️ Hoisting

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before executing any code. When JS runs your file, it first scans for all variable and function declarations, then executes the code line by line. This means you can sometimes use things before they appear in your code — but the behavior differs between var, let, and const.

var Hoisting

  • The declaration is hoisted, but the value is NOT
  • Using it before the line gives undefined (not an error)
  • This is confusing and hides bugs

let/const Hoisting (TDZ)

  • Also hoisted, but placed in the "Temporal Dead Zone"
  • Using it before the line gives a ReferenceError
  • This is safer — it catches the mistake immediately
hoisting.js
// var — hoisted as undefined
console.log(x); // undefined (no error, but wrong value!)
var x = 5;

// let — Temporal Dead Zone (TDZ)
console.log(y); // ❌ ReferenceError: Cannot access 'y' before initialization
let y = 5;

// Functions — fully hoisted (declaration + body)
greet(); // ✅ works — "Hello!"
function greet() { console.log("Hello!"); }
🔍 Scoping

Scope determines where a variable can be accessed. Think of it as the "visibility" of a variable — some variables are visible everywhere, others are only visible inside a specific function or block.

🌍 Global Scope

  • Declared outside any function or block
  • Accessible from anywhere in the file
  • Use sparingly — polluting global scope causes naming conflicts and hard-to-find bugs

🔧 Function Scope

  • Declared inside a function
  • Only accessible within that function
  • All three keywords (var, let, const) respect function scope

📦 Block Scope

  • Inside any { } — if, for, while, etc.
  • let and const stay inside the block
  • var ignores blocks (leaks out!) — another reason to avoid it
scope.js
const global = "visible everywhere"; // global scope

function example() {
  const local = "only in this function"; // function scope

  if (true) {
    const blocked = "only in this if-block"; // block scope
    var leaked = "I escape the block!"; // function scope (var!)

    console.log(global);  // ✅ accessible
    console.log(local);   // ✅ accessible
    console.log(blocked); // ✅ accessible
  }

  console.log(blocked); // ❌ ReferenceError
  console.log(leaked);  // ✅ "I escape the block!" — var leaks!
}
Section 03

Data Types

JavaScript is dynamically typed — variables can hold any type of value, and the type can change.

🧩 Primitive Data Types

JavaScript has several built-in data types. The most common ones you'll work with daily are strings, numbers, booleans, undefined, and null. JavaScript is dynamically typed, meaning you don't need to declare what type a variable will hold — the type is determined automatically by the value you assign.

string

  • Text wrapped in quotes: "hello", 'world'
  • Template literals use backticks: `Hi ${name}`
  • Template literals allow embedded expressions and multi-line strings
  • Strings are immutable — you can't change individual characters

number

  • Integers and decimals: 42, 3.14, -10
  • Special values: Infinity, -Infinity, NaN
  • No separate integer/float types (unlike many other languages)
  • NaN means "Not a Number" — result of invalid math

boolean

  • Only two values: true or false
  • Result of comparisons: 5 > 3 → true
  • Used in conditions: if (isLoggedIn) { ... }
  • Truthy/falsy: some non-boolean values act as true or false

undefined & null

  • undefined — variable declared but never assigned a value
  • null — intentionally set to "no value" by the programmer
  • Both mean "nothing," but undefined is JS's default, null is your choice
  • typeof null returns "object" — a famous JS bug that's never been fixed
typeof — checking types
typeof "hello"   // "string"
typeof 42       // "number"
typeof true     // "boolean"
typeof undefined // "undefined"
typeof null     // "object" (historical bug)
typeof []       // "object" (arrays are objects)
typeof {}       // "object"
Section 04

Operators & Control Flow

How to compare values, combine conditions, and make your code take different paths.

⚙️ Operators

Comparison operators compare two values and return true or false. Logical operators combine multiple conditions together.

⚖️ Comparison Operators

  • === — strict equal (checks value AND type) — always use this
  • !== — strict not equal
  • > < >= <= — greater/less than
  • == — loose equal (type coercion — avoid)

🔗 Logical Operators

  • && (AND) — both conditions must be true
  • || (OR) — at least one condition must be true
  • ! (NOT) — flips true to false and vice versa
  • Short-circuit: && stops at first false, || stops at first true
operators.js
// Comparison
5 === 5    // true — same value, same type
5 === "5"  // false — different types (number vs string)
5 == "5"   // true — type coercion (AVOID this!)

// Logical
true && true   // true
true && false  // false
true || false  // true
!true         // false
⚠️ Always use === (strict equality), never == (loose equality). Loose equality does type coercion, which leads to bizarre results: 0 == "" is true, null == undefined is true. Strict equality is predictable and safe.
🔀 Control Flow — if / else

Control flow lets your code make decisions. The most basic form is if / else — if a condition is true, run this code; otherwise, run that code.

control-flow.js
const score = 85;

// if / else if / else
if (score >= 90) {
  console.log("Excellent!");
} else if (score >= 70) {
  console.log("Good job!"); // ← this runs
} else {
  console.log("Keep trying!");
}

// Ternary operator (shorthand for simple if/else)
const status = score >= 70 ? "pass" : "fail";
// status → "pass"
Section 05

Functions

Reusable blocks of code that take input, do something, and return output.

🔧 Ways to Declare Functions

A function is a reusable block of code. You define it once, then call it whenever you need it. JavaScript offers several ways to write functions — each with slightly different behavior.

Function Declaration

  • Uses the function keyword with a name
  • Hoisted — can be called before it's defined
  • Traditional, widely understood syntax

Function Expression

  • Stored in a variable (often const)
  • NOT hoisted — must be defined before use
  • Can be anonymous (no name) or named

Arrow Function (ES6)

  • Short syntax using =>
  • NOT hoisted (same as expressions)
  • The standard in modern JS and React
functions.js
// 1. Function Declaration (named, hoisted)
function greet(name) {
  return `Hello, ${name}!`;
}

// 2. Function Expression (anonymous, stored in variable)
const greet = function(name) {
  return `Hello, ${name}!`;
};

// 3. Arrow Function (modern, concise)
const greet = (name) => `Hello, ${name}!`;

// Arrow function shortcuts:
const double = n => n * 2;       // single param: no parens needed
const sayHi = () => "Hi!";     // no params: empty parens required
const add = (a, b) => a + b;   // multiple params: parens required
🎯 In this program, we'll primarily use arrow functions — they're the standard in React and modern JavaScript projects. Learn all three forms so you can read any codebase, but default to arrows in your own code.
Section 06

Loops

Loops let you repeat code without writing the same thing over and over.

🔄 Types of Loops

Loops run a block of code repeatedly until a condition is met. JavaScript has several loop types — each suited for different situations.

for loop — when you know the count
// for (start; condition; step)
for (let i = 0; i < 5; i++) {
  console.log(i); // 0, 1, 2, 3, 4
}

// Looping through an array by index
const names = ["Karma", "Dorji", "Tshering"];
for (let i = 0; i < names.length; i++) {
  console.log(names[i]);
}
while & do...while
// while — runs while condition is true
let count = 0;
while (count < 3) {
  console.log(count); // 0, 1, 2
  count++;
}

// do...while — runs at least once, then checks
let n = 10;
do {
  console.log(n); // 10 (runs once even though condition is false)
  n++;
} while (n < 5);
for...of & for...in
// for...of — loop over array VALUES
const cities = ["Gelephu", "Thimphu", "Paro"];
for (const city of cities) {
  console.log(city); // "Gelephu", "Thimphu", "Paro"
}

// for...in — loop over object KEYS
const person = { name: "Karma", age: 25, city: "Gelephu" };
for (const key in person) {
  console.log(key, person[key]);
  // "name" "Karma", "age" 25, "city" "Gelephu"
}

When to Use Which Loop

LoopBest ForExample Use Case
forWhen you know exactly how many times to loopRun something 10 times, process by index
whileWhen you don't know when to stopKeep reading input until user types "quit"
do...whileMust execute at least once before checkingShow a menu, then ask "continue?"
for...ofLooping over array valuesProcess each name in a list
for...inLooping over object keysRead each property of a config object
💡 Pro tip: In modern JavaScript, you'll often use array methods like .forEach(), .map(), and .filter() instead of traditional loops. We'll cover those in the next class — they're more expressive and less error-prone.
