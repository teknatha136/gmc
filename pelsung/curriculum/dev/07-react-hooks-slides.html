04 What is a hook?
In one sentence

A hook is a function that adds a feature to your component

Mental model
A plain function returns JSX.
A function with hooks can also remember things and do things like fetch data, set timers, etc.

React gives you many built-in hooks. Today we focus on two: useState and useEffect.

05 One rule to remember

Always call hooks at the top of your component

Don't do this

function App() { if (isOpen) { const [x, setX] = useState(0); } … }

Do this

function App() { const [x, setX] = useState(0); if (isOpen) { … } … }

Never put hooks inside if, loops, or other functions. Just call them at the top, in the same order, every render.

06 Spot a hook

If the name starts with use, it's a hook

useState
Built-in. Remember a value.
useEffect
Built-in. Run code after render.

useState

A quick refresher and a few common shapes.

02
08 useState — quick recap
From Class 06

Two things back, one thing in

counter.tsx
const [count, setCount] = useState(0); // ↑ value ↑ setter ↑ initial

Calling setCount(...) updates the value and re-renders the component with the new value.

09 Common shapes

Numbers, strings, booleans, arrays

examples.tsx
// Numbers — counters, prices const [count, setCount] = useState(0); // Strings — input values, names const [name, setName] = useState(""); // Booleans — toggles, modal open/close const [isOpen, setOpen] = useState(false); // Arrays — lists of things const [events, setEvents] = useState<GmcEvent[]>([]);

TypeScript figures out the type from the initial value. For arrays of objects, write the type with <…>.

10 useState with an input

Type something — see it appear

name-box.tsx
function NameBox() { const [name, setName] = useState(""); return ( <div> <input value={name} onChange={e => setName(e.target.value)} /> <p>Hello, {name || "friend"}!</p> </div> ); }

Each keystroke updates state → React re-renders → the greeting updates. That's the whole loop.

11 Updating arrays — one rule
Important

Don't change the array — make a new one

Won't update the screen

events.push(newEvent); setEvents(events);

Make a new array

setEvents([...events, newEvent]);

Add with spread [...]. Remove with .filter(). Always pass a fresh array to the setter.

12 The simple loop

State changes → component re-runs → screen updates

Remember this
You change count with setCount.
React runs your component again with the new value.
The new JSX shows on screen.

That's it — that's the whole magic of useState.

useEffect

Run code after render — usually to load data.

03
21 Why we need useEffect

Some things shouldn't happen during render

The idea
Rendering = describing what the screen looks like.
useEffect = "after the screen is drawn, also do this."
The most common this is fetching data from the internet.
22 The shape of useEffect

A function and a list

shape.tsx
useEffect(() => { // the code that runs AFTER the render }, [/* the list of things to watch */]);

Two things: a function (what to do), and an array (when to do it).

23 The list — two simple patterns
List looks like…Effect runs…Use for
[]Once, when the component appearsLoading data on first show
[id]Once, plus every time id changesRe-loading when something changes

Today we'll only use the empty list [] — load once, when the page opens. That's enough to do a lot.

24 Live demo — fetch from the internet
Trainer demo

A free practice API: JSONPlaceholder

posts.tsx
import { useEffect, useState } from "react"; type Post = Readonly<{ id: number; title: string; body: string }>; export function Posts() { const [posts, setPosts] = useState<Post[]>([]); useEffect(() => { fetch("https://jsonplaceholder.typicode.com/posts") .then(res => res.json()) .then(data => setPosts(data)); }, []); return ( <ul className="space-y-2"> {posts.map(p => <li key={p.id}>{p.title}</li>)} </ul> ); }

fetch sends a request. .then(res => res.json()) gets the data. setPosts stores it.

25 Show "Loading…" while waiting
posts.tsx — with loading
const [posts, setPosts] = useState<Post[]>([]); const [loading, setLoading] = useState(true); useEffect(() => { fetch("https://jsonplaceholder.typicode.com/posts") .then(res => res.json()) .then(data => { setPosts(data); setLoading(false); }); }, []); if (loading) return <p className="text-gray-500">Loading…</p>; return <ul className="space-y-2">{posts.map(…)}</ul>;

Two pieces of state — posts and loading. While waiting, show a message.

35 What is a custom hook?

A function that uses other hooks inside it

Mental model
You wrote the same useState + setX logic in two components?
Move it into one function called useX.
Both components call it. No duplication.

One rule: the function name must start with use.

36 Example — useCounter
use-counter.ts
import { useState } from "react"; export function useCounter(start: number = 0) { const [count, setCount] = useState(start); const add = () => setCount(count + 1); const remove = () => setCount(count > 0 ? count - 1 : 0); const reset = () => setCount(start); return { count, add, remove, reset }; }

One file. Used in any component as const { count, add, reset } = useCounter(0);.

37 Using it
events-counter.tsx — refactored
import { useCounter } from "./use-counter"; export function EventsCounter() { const { count, add, remove, reset } = useCounter(0); return ( <div className="flex flex-col gap-3 p-4"> <p className="text-lg">Scheduled: {count}</p> <div className="flex gap-2"> <button className="rounded-md bg-blue-500 px-3 py-1.5 text-white" onClick={add}>+ Add</button> <button className="rounded-md border border-gray-300 px-3 py-1.5" onClick={remove}>− Remove</button> <button className="rounded-md border border-gray-300 px-3 py-1.5" onClick={reset}>Reset</button> </div> </div> ); }

The component shrinks to almost nothing. The logic lives once, in use-counter.ts.
