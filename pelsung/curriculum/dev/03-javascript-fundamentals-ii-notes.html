Section 01

Objects — grouping related data

Objects let you bundle many pieces of information about one thing into a single container.

📦 What is an object?

An object is a collection of key–value pairs. Think of it like a labelled box with many compartments — each compartment has a name (the key) and holds a value.

object-basics.js
// An object with four properties
const trainee = {
  name: "Karma",
  age: 24,
  vertical: "Dev",
  isActive: true
};

Vocabulary

Key

  • The label you use to find the value
  • e.g. name, age
  • Must be unique within the object

Value

  • What's stored under that key
  • Can be any type: string, number, boolean, array, another object, even a function

Property

  • A key + value pair together
  • name: "Karma" is one property
🎯 Accessing properties

There are two ways to read a value from an object: dot notation and bracket notation. Most of the time you'll use dot notation — it's cleaner.

accessing.js
const trainee = { name: "Karma", age: 24 };

// Dot notation — most common
console.log(trainee.name); // "Karma"
console.log(trainee.age);  // 24

// Bracket notation — useful when the key is a variable
const key = "name";
console.log(trainee[key]); // "Karma"

// Also needed if the key has special characters
const settings = { "dark-mode": true };
console.log(settings["dark-mode"]); // true
💡 Rule of thumb: Use dot notation by default. Reach for bracket notation only when the key is stored in a variable or contains special characters like spaces or dashes.
✏️ Adding, updating, deleting

Objects are flexible. You can add new properties after creation, update existing ones, or remove them entirely — even if the variable is declared with const.

modify.js
const trainee = { name: "Karma", age: 24 };

// Add
trainee.vertical = "Dev";

// Update
trainee.age = 25;

// Delete
delete trainee.age;

console.log(trainee);
// { name: "Karma", vertical: "Dev" }
⚠️ Why does const still let us change the object? const locks the variable (so you can't point it at a different object) but not the contents. The object itself is mutable.
⚙️ Methods — functions as values

When a property's value is a function, we call it a method. Methods let objects do things, not just hold data.

methods.js
const trainee = {
  name: "Karma",
  introduce() {
    return `Hi, I'm ${this.name}`;
  }
};

console.log(trainee.introduce()); // "Hi, I'm Karma"

Inside a method, the word this refers to the object the method belongs to — so this.name means "the name of this trainee." We'll go deeper into this in later classes.

🪆 Nested objects

A value inside an object can itself be another object. This is how real data is shaped — a user has an address, which has a city, which has a district, and so on.

nested.js
const trainee = {
  name: "Karma",
  address: {
    city: "Thimphu",
    country: "Bhutan"
  }
};

console.log(trainee.address.city); // "Thimphu"
🌐 JSON — data from the backend

Most of the time, you won't write objects by hand. You'll fetch them from a server — a backend API. The server sends data as JSON (JavaScript Object Notation), which is just text that looks like a JS object.

When your code receives a JSON response, JavaScript automatically turns it into a regular object, and you read it the same way you've been reading any other object.

api-response.json
// What the server sends when you call GET /api/trainees/42
{
  "id": 42,
  "name": "Karma Dorji",
  "vertical": "Dev",
  "isActive": true,
  "contact": {
    "email": "karma@example.com",
    "phone": "+975-17-000000"
  },
  "skills": ["HTML", "CSS", "JS"]
}
🔑 Key insight: JSON keys are always wrapped in double quotes — that's the main visual difference from JS object literals. Once parsed into JavaScript, it's just an object: trainee.contact.email, trainee.skills[0], etc.

Objects — takeaways

  • Objects group related data as key–value pairs
  • Access values with dot notation (user.name) — fallback to bracket for dynamic keys
  • You can add, update, and delete properties even on a const object
  • Functions stored as values are called methods; this refers to the object
  • Most real-world data comes from APIs as JSON — it's just objects under the hood
Section 02

Arrays — ordered lists

When you have many of the same kind of thing (a list of trainees, scores, tasks), use an array.

📋 What is an array?

An array is an ordered, numbered list. Each item has a position called an index, starting at 0.

arrays.js
const fruits = ["apple", "banana", "mango"];
// ↑ 0 ↑ 1 ↑ 2

console.log(fruits[0]); // "apple"
console.log(fruits[2]); // "mango"
console.log(fruits.length); // 3

// Arrays can hold any type, including mixed
const mixed = [1, "hello", true, { name: "Karma" }];
📌 Zero-based indexing. The first item is at index 0, not 1. The last item is at index length - 1. This catches beginners out — always count from zero.
➕ Adding & removing

Four methods handle adding and removing items at the ends of an array. They all modify the original array in place.

MethodWhereWhat it does
push(item)EndAdds an item to the end
pop()EndRemoves the last item and returns it
unshift(item)StartAdds an item to the beginning
shift()StartRemoves the first item and returns it
add-remove.js
const fruits = ["apple", "banana"];

fruits.push("mango"); // ["apple", "banana", "mango"]
fruits.pop(); // returns "mango" — array: ["apple", "banana"]
fruits.unshift("kiwi"); // ["kiwi", "apple", "banana"]
fruits.shift(); // returns "kiwi" — array: ["apple", "banana"]
🔍 Searching

When you need to check if a value exists, find its position, or locate the first item matching some rule, use one of these methods.

MethodReturnsUse case
includes(item)true or false"Does this array contain X?"
indexOf(item)Index (or -1)"At what position is X?"
find(fn)First matching item (or undefined)"Give me the first trainee where X"
findIndex(fn)Index of first match (or -1)Same, but I want the position
searching.js
[1, 2, 3].includes(2); // true
["a", "b", "c"].indexOf("b"); // 1
["a", "b"].indexOf("z"); // -1 (not found)

const trainees = [
  { name: "Karma", score: 82 },
  { name: "Sonam", score: 91 }
];
trainees.find(t => t.score > 85);
// { name: "Sonam", score: 91 }
Section 03

Array iteration — map, filter, reduce

These three methods are the heart of modern JavaScript. Learn them well — you'll use them every day.

👟 forEach — do something with each item

forEach runs a function once for every item in the array. It does not return anything. Use it when you want to do something (log, send, print) for each item.

forEach.js
const names = ["Karma", "Sonam", "Tshering"];

names.forEach(name => {
  console.log(`Hello ${name}!`);
});
// Hello Karma! / Hello Sonam! / Hello Tshering!
🔄 map — transform every item

map produces a new array of the same length, where each item is transformed by your function. The original array is not changed.

map.js
const scores = [82, 91, 74];

// Transform numbers
const doubled = scores.map(s => s * 2);
// [164, 182, 148]

// Extract one field from each object
const trainees = [{ name: "Karma" }, { name: "Sonam" }];
const names = trainees.map(t => t.name);
// ["Karma", "Sonam"]
🔍 filter — keep only what matches

filter produces a new array containing only the items where your function returns true. The result may be shorter than the original.

filter.js
const scores = [82, 41, 91, 55, 74];

const passing = scores.filter(s => s >= 60);
// [82, 91, 74]

const trainees = [
  { name: "Karma", vertical: "Dev" },
  { name: "Sonam", vertical: "QA" }
];
const devs = trainees.filter(t => t.vertical === "Dev");
📉 reduce — combine all items into one value

reduce "rolls up" an array into a single value — a sum, an average, a combined string, a count, etc. It takes two things: a function and a starting value.

reduce.js
const scores = [82, 91, 74];

const total = scores.reduce((sum, s) => sum + s, 0);
// ↑ starting value
// Iteration 1: sum=0, s=82 → returns 82
// Iteration 2: sum=82, s=91 → returns 173
// Iteration 3: sum=173, s=74 → returns 247
// total = 247
💡 Getting comfortable with reduce takes practice. Just remember: the first parameter (sum) is the "running total so far." Whatever you return becomes the next iteration's running total.
📊 sort — reorder items

sort reorders the array. For strings it works out of the box. For numbers there's a famous gotcha — you must pass a comparator function.

sort.js
// Strings — works alphabetically
["Sonam", "Karma", "Tshering"].sort();
// ["Karma", "Sonam", "Tshering"]

// Numbers — default sort is wrong!
[10, 2, 30, 4].sort();
// [10, 2, 30, 4] ← sorted as strings: "10" < "2" < "30" < "4"

// Fix: pass a comparator
[10, 2, 30, 4].sort((a, b) => a - b);
// [2, 4, 10, 30] ✓
⚠️ Two things to know. First, sort() modifies the original array (unlike map/filter). Second, always pass (a, b) => a - b for numbers — otherwise they get sorted alphabetically.
✂️ slice vs splice

Two methods with almost the same name, but very different behavior.

MethodWhat it doesModifies original?Returns
slice(start, end)Makes a copy of a sectionNoNew array
splice(start, count)Removes items (can also insert)YesArray of removed items
slice-splice.js
const nums = [1, 2, 3, 4, 5];

nums.slice(1, 3); // [2, 3] — nums unchanged
nums.splice(1, 2); // removes [2,3] — nums is now [1, 4, 5]

Arrays — takeaways

  • Indexes start at 0; arr.length tells you how many items
  • push/pop add/remove at the end; unshift/shift at the start
  • map transforms, filter selects, reduce combines — these are the workhorses
  • find returns the first match; filter returns all matches
  • sort modifies the original — always pass a comparator for numbers
  • slice is safe (returns a copy); splice changes the original
Section 04

Destructuring — unpack values cleanly

A short syntax for pulling values out of objects and arrays. Makes code much easier to read.

📦 Object destructuring

Instead of writing the same object name over and over to pull out its properties, destructuring lets you do it in one line.

object-destructure.js
const trainee = { name: "Karma", age: 24, vertical: "Dev" };

// Old way
const name = trainee.name;
const age = trainee.age;

// Destructured
const { name, age, vertical } = trainee;
console.log(name); // "Karma"
console.log(vertical); // "Dev"

Rename and default values

rename-defaults.js
const trainee = { name: "Karma", age: 24 };

// Rename as you destructure
const { name: traineeName } = trainee;
console.log(traineeName); // "Karma"

// Default — used only if the key is missing
const { city = "Thimphu" } = trainee;
console.log(city); // "Thimphu" (trainee has no city)
🪆 Nested destructuring

You can reach into nested objects the same way they were built. This is particularly handy with API responses.

nested-destructure.js
const trainee = {
  name: "Karma",
  contact: { email: "karma@x.com", phone: "1234" }
};

const { contact: { email } } = trainee;
console.log(email); // "karma@x.com"
📋 Array destructuring

Arrays are destructured by position, not name. You list variable names in brackets, and they grab the items in order.

array-destructure.js
const colors = ["red", "green", "blue"];

// Basic
const [first, second] = colors;
console.log(first); // "red"
console.log(second); // "green"

// Skip items using commas
const [, , third] = colors;
console.log(third); // "blue"

// "Rest" — collect remaining items
const [head, ...tail] = colors;
// head = "red", tail = ["green", "blue"]
📤 Spread ... — copy & combine

The same three dots used for "rest" can also be used to spread values out — copying arrays, merging objects, or passing list items as function arguments.

spread.js
// Copy & extend an array
const colors = ["red", "green"];
const extended = [...colors, "blue"];
// ["red", "green", "blue"]

// Merge objects — later keys overwrite earlier ones
const base = { name: "Karma", age: 24 };
const updated = { ...base, age: 25 };
// { name: "Karma", age: 25 }
Section 05

Optional chaining — safe property access

How to read deep into an object without crashing when part of the path is missing.

🛡️ The ?. operator

Real data from APIs is often incomplete. A user might not have filled in their address; a response might skip optional fields. Normal property access crashes when something in the chain is undefined or null. Optional chaining (?.) is a safety net.

optional-chaining.js
const trainee = { name: "Karma" };
// Notice: no "contact" field

// ❌ Without optional chaining — crashes
console.log(trainee.contact.email);
// TypeError: Cannot read properties of undefined (reading 'email')

// ✓ With optional chaining — safe
console.log(trainee.contact?.email);
// undefined (no crash — just "nothing there")

// Combine with ?? to supply a fallback
const email = trainee.contact?.email ?? "no email";
console.log(email); // "no email"
🎯 When to use it: any time you're accessing a property that might not exist. Most common with API responses, user input, and deeply nested data.

You can chain multiple ?. together. If anywhere in the chain is missing, the whole expression becomes undefined instead of throwing an error.

deep-chaining.js
const user = { profile: { address: null } };

user.profile?.address?.city?.zipCode;
// undefined — no crash, even though address is null

Destructuring & ?. — takeaways

  • Destructuring unpacks values from objects and arrays in one line
  • Object destructuring uses names; array destructuring uses positions
  • You can rename ({ name: x }) and provide defaults ({ city = "Thimphu" })
  • ... means "rest" on the left side, "spread" on the right — opposite operations, same syntax
  • Use ?. to safely access keys that might not exist — no more unexpected crashes
  • Pair ?. with ?? to provide a fallback value
