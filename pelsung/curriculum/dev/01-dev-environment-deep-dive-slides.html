04 JavaScript
What is JavaScript?

The Language That Powers the Web

🌐

Born in the Browser

Created in 1995 by Brendan Eich at Netscape. Originally designed to make web pages interactive.

⚡

Runs Everywhere

Every browser has a JavaScript engine. Chrome uses V8, Firefox uses SpiderMonkey, Safari uses JavaScriptCore.

🏗️

Not Just Browsers

Today JS also runs on servers (Node.js), mobile apps (React Native), and desktop apps (Electron).

🎯 Key insight: JavaScript is the only programming language that every web browser understands natively. That's why it dominates web development.

05 JavaScript Today

What JavaScript Can Do

  • Frontend: Build interactive web pages — buttons, animations, forms, everything you see and click
  • Backend: Run server-side code with Node.js — handle requests, talk to databases
  • Mobile: Build mobile apps with React Native — one language for iOS and Android
  • Desktop: Build desktop apps with Electron — VS Code, Slack, and Discord are all JavaScript
  • AI tools: Claude Code's interface, ChatGPT's frontend, most AI tools use JavaScript on the client side

07 Node.js
Node.js

JavaScript Outside the Browser

Node.js lets you run JavaScript on your computer — no browser needed

📝 You write JS app.js
→
⚙️ Node.js V8 engine
→
💻 Your Computer Runs the code

🔧 What It Is

  • A runtime environment for JavaScript
  • Built on Chrome's V8 engine (same engine Chrome uses)
  • Created by Ryan Dahl in 2009
  • Free and open source

🧠 Think of It Like

  • JS in browser = fish in an aquarium (limited)
  • JS with Node = fish in the ocean (free to do anything)
  • Node gives JS access to your file system, network, OS
  • That's why we can build servers with it
08 Why Node.js?

Why Do We Need Node.js?

  • One language everywhere: Write JavaScript for both frontend and backend — no need to learn two languages
  • npm ecosystem: Access to 2+ million free packages — someone already solved your problem
  • Dev tooling: Almost every modern web development tool runs on Node — Vite, React, ESLint, Prettier
  • Industry standard: Netflix, PayPal, LinkedIn, Uber all use Node.js in production
  • Fast to get started: Install Node → write a .js file → run it. That's it. No complex setup needed.

09 Before vs After

Web Dev Before & After Node.js

❌ Before Node.js

  • Frontend: JavaScript
  • Backend: PHP, Java, Ruby, Python
  • Learn 2+ languages to build a web app
  • Different tools, syntax, ecosystems
  • Hard to share code between FE & BE

✅ After Node.js

  • Frontend: JavaScript
  • Backend: JavaScript (Node.js)
  • One language for the entire stack
  • Share code, libraries, knowledge
  • Massive ecosystem via npm

11 npm Basics
npm

What is npm?

The world's largest software registry — 2+ million packages

📦

Package Registry

A massive online library of free, reusable code that anyone can publish and download.

🛠️

Command-Line Tool

Comes bundled with Node.js. You use it in the terminal to install, update, and manage packages.

📋

package.json

Every project has one. It lists all your dependencies, scripts, and project metadata.

12 package.json

Anatomy of package.json

The ID card of every Node.js project

package.json
{
  "name": "my-first-project",
  "version": "1.0.0",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "start": "node server.js"
  },
  "dependencies": {
    "react": "^18.2.0",
    "axios": "^1.6.0"
  }
}
13 npm Commands

Essential npm Commands

npm init Create a new project (generates package.json)
npm install Install all dependencies listed in package.json
npm install react Add a specific package to your project
npm run dev Run the "dev" script from package.json
npm run build Run the "build" script (create production files)
npm start Run the "start" script (launch the app)

💡 node_modules/ — when you run npm install, all downloaded packages go here. Never edit or commit this folder.

Terminal

Why Use the Terminal?

🖱️ GUI (What You Know)

  • Click folders to open them
  • Drag files to move them
  • Right-click for options
  • Good for browsing, slow for dev work

⌨️ Terminal (What Devs Use)

  • Type commands to do things
  • Much faster once you learn it
  • Can automate repetitive tasks
  • Required for git, npm, Node.js, deployments

🎯 Every professional developer uses the terminal daily. It's not optional — it's a core skill.

16 Navigation

Navigation & Files

Moving around and managing files

pwd Print working directory — "where am I right now?"
ls List files and folders in current directory
cd folder Change directory — move into a folder
cd .. Go up one level (back to parent folder)
mkdir name Make a new directory (folder)
touch file.js Create a new empty file
rm file.js Delete a file (careful — no undo!)
clear Clear the terminal screen
17 Dev Commands

Dev Workflow Commands

Commands you'll use every day as a developer

node file.js Run a JavaScript file with Node.js
npm run dev Start the development server
npm install Install project dependencies
git status Check what files have changed
code . Open current folder in VS Code
Ctrl + C Stop a running process (e.g., dev server)
