Variables

What is a Variable?

A labeled container that holds a value

variables.js
// Three ways to declare a variable
var name = "Karma";    // old way — avoid
let age = 25;         // can be reassigned
const city = "Gelephu"; // cannot be reassigned

// Reassignment
age = 26;             // ✅ works — let allows this
city = "Thimphu";      // ❌ error — const prevents this

🎯 Rule of thumb: Use const by default. Use let only when the value needs to change. Never use var.

05 Comparison

var vs let vs const

Feature var let const
Can be reassigned? Yes Yes No
Can be re-declared? Yes No No
Scope Function Block Block
Hoisted? Yes (as undefined) Yes (TDZ) Yes (TDZ)
Use in 2026? ❌ Avoid ✅ When value changes ✅ Default choice
06 Why Not var?

Why We Avoid var

var — surprising behavior
// var leaks out of blocks
if (true) {
  var x = 10;
}
console.log(x); // 10 😱

// var can be re-declared
var name = "Karma";
var name = "Dorji";
// No error — silent bug!
let — predictable behavior
// let stays in its block
if (true) {
  let x = 10;
}
console.log(x); // ❌ Error

// let prevents re-declaration
let name = "Karma";
let name = "Dorji";
// ❌ Error — bug caught!
07 When to Use Each

const vs let in Practice

examples.js
// ✅ const — value won't change
const API_URL = "https://api.example.com";
const MAX_RETRIES = 3;
const user = { name: "Karma", age: 25 };

// ✅ let — value will change
let count = 0;
count = count + 1;

let isLoggedIn = false;
isLoggedIn = true;

// ⚠️ const with objects — the reference is fixed,
// but properties can change
user.age = 26; // ✅ works!
user = {};     // ❌ error

Hoisting

JavaScript Reads Your Code Twice

First pass: find all declarations. Second pass: execute the code.

var hoisting
console.log(x); // undefined
var x = 5;
console.log(x); // 5

// JS sees this as:
var x;          // hoisted
console.log(x); // undefined
x = 5;          // assigned
console.log(x); // 5
let/const hoisting (TDZ)
console.log(y); // ❌ ReferenceError
let y = 5;

// let & const are hoisted too,
// but they stay in the
// "Temporal Dead Zone" (TDZ)
// until the declaration line.

// This is SAFER — it catches
// bugs where you use a variable
// before declaring it.
10 Scoping
Scoping

Where Can You Access a Variable?

🌍 Global Scope

  • Declared outside any function or block
  • Accessible from anywhere
  • Use sparingly — can cause bugs

🔧 Function Scope

  • Declared inside a function
  • Only accessible within that function
  • var, let, and const all respect this

📦 Block Scope

  • Inside { } — if, for, while blocks
  • let and const stay inside the block
  • var ignores blocks (leaks out!)
11 Scope Examples

Scope in Action

scope.js
const global = "I'm everywhere"; // global

function greet() {
  const local = "I'm only here"; // function scope

  if (true) {
    const inner = "I'm trapped"; // block scope
    console.log(global); // ✅
    console.log(local);  // ✅
    console.log(inner);  // ✅
  }

  console.log(inner); // ❌ not defined
}

console.log(local); // ❌ not defined

What kind of values can JavaScript hold?

03
13 Data Types
Primitive Types

JavaScript's Built-in Data Types

string

  • "Hello", 'World', `Hi ${name}`
  • Text values wrapped in quotes
  • Template literals use backticks

number

  • 42, 3.14, -10, Infinity
  • Integers and decimals — same type
  • No separate int/float distinction

boolean

  • true or false
  • Used in conditions and logic
  • Result of comparisons: 5 > 3 → true

undefined

  • Variable declared but not assigned
  • let x; → x is undefined
  • JS's default "no value yet"

null

  • Intentionally empty — "no value"
  • let result = null;
  • You set this deliberately

typeof

  • typeof "hi" → "string"
  • typeof 42 → "number"
  • Check a value's type at runtime

15 Operators

JavaScript Operators

🔢 Arithmetic

  • + — addition
  • - — subtraction
  • * — multiplication
  • / — division
  • % — modulo (remainder)
  • ++ / -- — increment / decrement

⚖️ Comparison

  • === — strict equal (use this!)
  • !== — strict not equal
  • > < >= <= — greater/less than
  • == — loose equal (avoid!)

🔗 Logical

  • && — AND: both must be true
  • || — OR: at least one must be true
  • ! — NOT: flips true ↔ false
operators.js
10 + 3    // 13     10 - 3   // 7     10 * 3   // 30
10 / 3    // 3.33...   10 % 3   // 1 (remainder)

5 === "5"   // false — different types
5 == "5"    // true — type coercion (avoid!)

true && false  // false    true || false  // true    !true  // false
16 Control Flow

if / else — Making Decisions

if / else if / else
const score = 85;

if (score >= 90) {
  console.log("Excellent!");
} else if (score >= 70) {
  console.log("Good job!");
} else {
  console.log("Keep trying!");
}

// Output: "Good job!"
ternary (shorthand)
// condition ? ifTrue : ifFalse

const age = 20;

const status = age >= 18
  ? "adult"
  : "minor";

// status → "adult"

// Great for simple conditions.
// Use if/else for complex ones.

Functions

Reusable blocks of code — the building blocks of every program

05
18 Function Types

Ways to Write Functions

Function Declaration (named)
// Named function — hoisted
function greet(name) {
  return `Hello, ${name}!`;
}

greet("Karma");
// "Hello, Karma!"
Function Expression (anonymous)
// Stored in a variable — not hoisted
const greet = function(name) {
  return `Hello, ${name}!`;
};

greet("Dorji");
// "Hello, Dorji!"
19 Arrow Functions

Arrow Functions (ES6)

Modern, concise syntax — the most common style in React

arrow-functions.js
// Standard arrow function
const greet = (name) => {
  return `Hello, ${name}!`;
};

// Short form — implicit return (one expression)
const greet = (name) => `Hello, ${name}!`;

// Single param — parentheses optional
const double = n => n * 2;

// No params — empty parentheses required
const sayHi = () => "Hi!";

// Multiple params
const add = (a, b) => a + b;

🎯 In this program: We'll use arrow functions most of the time — they're the standard in React and modern JavaScript.

Loops

Repeating code without repeating yourself

06
21 Loop Types

Types of Loops

for loop
// Classic for loop
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// 0, 1, 2, 3, 4
while loop
// Runs while condition is true
let count = 0;
while (count < 3) {
  console.log(count);
  count++;
}
// 0, 1, 2
do...while loop
// Runs at least once
let n = 0;
do {
  console.log(n);
  n++;
} while (n < 3);
// 0, 1, 2
for...of loop
// Loop over arrays
const cities = ["Gelephu",
  "Thimphu", "Paro"];

for (const city of cities) {
  console.log(city);
}
22 Which Loop?

When to Use Which Loop

Loop Best For Example
for When you know the exact count Run 10 times, loop through indices
while When you don't know when to stop Keep asking until user says "quit"
do...while Must run at least once Show menu, then ask to continue
for...of Looping over arrays Process each item in a list
for...in Looping over object keys Read each property of an object
