04 Why today matters
Context

Real data is not just numbers and strings

When you build real apps, data comes in groups — a user has a name, email, age, address. A shopping cart has many items. A list of trainees has 300 entries.

Today we learn the two tools that handle all of this: objects and arrays.

Section 01

Objects

How to group related data together

01
06 Objects
Concept

What is an object?

📦 Think of it like this
An object is a box with labelled compartments. Each compartment holds one piece of information — and each compartment has a name so you can find it later.
object.js
// A trainee — one box, many pieces of info
const trainee = {
  name: "Karma",
  age: 24,
  vertical: "Dev",
  isActive: true
};
07 Objects
Vocabulary

Key · Value · Property

vocabulary.js
const trainee = {
  name: "Karma",
  age: 24
};
// ↑ key ↑ value
// "name: Karma" is one property
key
The label — the name you use to find the value. Always a string.
value
What's stored — can be any type (string, number, boolean, array, even another object).
property
A key + value pair together. name: "Karma" is one property.
08 Objects
Accessing values

Dot notation vs bracket notation

access.js
const trainee = { name: "Karma", age: 24 };

// Dot notation — most common, cleanest
console.log(trainee.name); // "Karma"
console.log(trainee.age);  // 24

// Bracket notation — when key is dynamic or has spaces
console.log(trainee["name"]); // "Karma"

const key = "age";
console.log(trainee[key]); // 24 — key is a variable
Rule of thumb: Use dot notation when you know the key name. Use bracket notation when the key is stored in a variable or has special characters.
09 Objects
Modifying

Add · Update · Delete

modify.js
const trainee = { name: "Karma", age: 24 };

// Add a new property
trainee.vertical = "Dev";

// Update an existing property
trainee.age = 25;

// Delete a property
delete trainee.age;

console.log(trainee);
// { name: "Karma", vertical: "Dev" }
💡 Notice: We used const but we're still modifying the object. That's fine — const prevents reassigning the variable, not changing what's inside.
10 Objects
Behavior

Methods — functions inside objects

When a value is a function, we call it a method.

methods.js
const trainee = {
  name: "Karma",
  age: 24,
  // a method — a function attached to the object
  introduce() {
    return `Hi, I'm ${this.name}`;
  }
};

console.log(trainee.introduce()); // "Hi, I'm Karma"

The word this means "the object the method is inside." We'll go deeper into this later — for now, just know it refers to the trainee.

11 Objects
Composition

Objects inside objects

A value can itself be another object. This is how real data is shaped.

nested.js
const trainee = {
  name: "Karma",
  address: {
    city: "Thimphu",
    country: "Bhutan"
  },
  scores: {
    js: 82,
    css: 74
  }
};

console.log(trainee.address.city); // "Thimphu"
console.log(trainee.scores.js);   // 82
12 Objects · Real world
Real world

Where do objects come from?

Most of the time, you don't write objects by hand. They come from a backend server — usually as JSON text.

JSON
JavaScript Object Notation — a text format for sending data between computers. Looks almost identical to JS objects.
API response
When you fetch data (e.g. from /users/42), the server sends back JSON, and JS turns it into an object.
13 Objects · JSON in the wild

A real API response

Imagine you called GET /api/trainees/42 — the server sends back this:

response.json
{
  "id": 42,
  "name": "Karma Dorji",
  "vertical": "Dev",
  "isActive": true,
  "contact": {
    "email": "karma@example.com",
    "phone": "+975-17-000000"
  },
  "skills": ["HTML", "CSS", "JS"]
}

Once in JS, it's just an object. You access it exactly the same way:
trainee.contact.email

Arrays

Ordered lists of many things

02
16 Arrays
Concept

What is an array?

📋 Think of it like this
An array is a numbered list. Each item has a position (called its index), starting from 0.
array.js
const fruits = ["apple", "banana", "mango"];
// ↑ 0 ↑ 1 ↑ 2

console.log(fruits[0]); // "apple"
console.log(fruits[2]); // "mango"
console.log(fruits.length); // 3

// Arrays can hold any type — numbers, objects, mixed
const scores = [82, 91, 74];
17 Arrays
Modifying

Adding & removing items

push(item)
Add to the end. Most common.
pop()
Remove from the end and return it.
unshift(item)
Add to the start. Shifts all others right.
shift()
Remove from the start and return it.
add-remove.js
const fruits = ["apple", "banana"];

fruits.push("mango");   // ["apple", "banana", "mango"]
fruits.pop();          // ["apple", "banana"] — returns "mango"
fruits.unshift("kiwi"); // ["kiwi", "apple", "banana"]
fruits.shift();        // ["apple", "banana"] — returns "kiwi"
18 Arrays
Searching

Finding things in an array

includes(item)
Returns true or false. "Is this value in the array?"
indexOf(item)
Returns the index of the item, or -1 if not found.
find(fn)
Returns the first item that matches your test function.
findIndex(fn)
Like find, but returns the item's index instead.
search.js
const trainees = [{name: "Karma", age: 24}, {name: "Sonam", age: 26}];

trainees.find(t => t.name === "Sonam");
// { name: "Sonam", age: 26 }

[1, 2, 3].includes(2); // true
["a", "b", "c"].indexOf("b"); // 1
19 Arrays · Iteration
Iteration · 1/4

forEach — do something with each item

Runs a function once for every item. Does not return anything new.

forEach.js
const names = ["Karma", "Sonam", "Tshering"];

names.forEach(name => {
  console.log(`Hello ${name}!`);
});

// Hello Karma!
// Hello Sonam!
// Hello Tshering!

Use forEach when you just want to do something (print, send a message) — not build a new list.

20 Arrays · Iteration
Iteration · 2/4

map — transform every item

Makes a new array where each item is transformed.

map.js
const scores = [82, 91, 74];

const doubled = scores.map(s => s * 2);
// [164, 182, 148]

const trainees = [{name: "Karma"}, {name: "Sonam"}];
const names = trainees.map(t => t.name);
// ["Karma", "Sonam"]
Original array is not changed. map always returns a new array of the same length.
21 Arrays · Iteration
Iteration · 3/4

filter — keep only what matches

Makes a new (possibly shorter) array containing only the items that pass your test.

filter.js
const scores = [82, 41, 91, 55, 74];

const passing = scores.filter(s => s >= 60);
// [82, 91, 74]

const trainees = [
  {name: "Karma", vertical: "Dev"},
  {name: "Sonam", vertical: "QA"},
  {name: "Tshering", vertical: "Dev"}
];
const devs = trainees.filter(t => t.vertical === "Dev");
// [{Karma...}, {Tshering...}]
22 Arrays · Iteration
Iteration · 4/4

reduce — combine all items into one value

Takes the array and "rolls it up" into a single value — a sum, a product, a combined string, anything.

reduce.js
const scores = [82, 91, 74];

const total = scores.reduce((sum, s) => sum + s, 0);
// ↑ starting value
// total = 82 + 91 + 74 = 247

const average = total / scores.length; // 82.3
💡 reduce feels tricky at first. Just remember: sum is the running total, s is the current number, and you return the new total each time.
23 Arrays
Sorting

sort — reorder items

sort.js
// Strings — sorts alphabetically
["Sonam", "Karma", "Tshering"].sort();
// ["Karma", "Sonam", "Tshering"]

// Numbers — ⚠️ surprise! Sort treats them as strings by default
[10, 2, 30, 4].sort();
// [10, 2, 30, 4] ← wrong!

// Fix: pass a comparator function
[10, 2, 30, 4].sort((a, b) => a - b);
// [2, 4, 10, 30] ✓
⚠️ Gotcha: sort() modifies the original array (unlike map/filter). Also — always pass a comparator for numbers.
24 Arrays
Commonly confused

slice vs splice

MethodWhat it doesModifies original?
slice(start, end) Returns a copy of a section No
splice(start, count) Removes items (can also insert) Yes
slice-splice.js
const nums = [1, 2, 3, 4, 5];

nums.slice(1, 3);   // [2, 3] — nums is still [1,2,3,4,5]
nums.splice(1, 2);  // removes [2,3] — nums is now [1, 4, 5]

Destructuring

Unpack values cleanly · plus ?. optional chaining

03
29 Destructuring
Motivation

The problem

Pulling values out of objects gets repetitive fast.

the-old-way.js
const trainee = { name: "Karma", age: 24, vertical: "Dev" };

// Old way — write the object name three times
const name = trainee.name;
const age = trainee.age;
const vertical = trainee.vertical;

// Destructuring — one line
const { name, age, vertical } = trainee;

Same result. Much cleaner.

30 Destructuring · Objects
Object destructuring

Rename & default values

destructure-object.js
const trainee = { name: "Karma", age: 24 };

// Basic — names must match the keys
const { name, age } = trainee;

// Rename on the way out
const { name: traineeName } = trainee;
console.log(traineeName); // "Karma"

// Default values — used when the key is missing
const { city = "Thimphu" } = trainee;
console.log(city); // "Thimphu" (trainee has no city)
31 Destructuring · Nested
Nested

Unpacking objects inside objects

nested-destructure.js
const trainee = {
  name: "Karma",
  contact: { email: "karma@x.com", phone: "1234" }
};

// Reach into contact and pull out email
const { contact: { email } } = trainee;
console.log(email); // "karma@x.com"

// Works great for API responses
32 Destructuring · Arrays
Array destructuring

Unpack by position

array-destructure.js
const colors = ["red", "green", "blue"];

// Basic — by position (not by name)
const [first, second] = colors;
console.log(first); // "red"

// Skip items with a comma
const [, , third] = colors;
console.log(third); // "blue"

// "Rest" — collect the remaining into one array
const [head, ...tail] = colors;
// head = "red", tail = ["green", "blue"]
33 Destructuring · Spread
Related idea

Spread ... — the opposite of rest

Same three dots, used to copy or combine instead of collect.

spread.js
// Copy an array
const colors = ["red", "green"];
const copy = [...colors, "blue"];
// ["red", "green", "blue"]

// Merge objects — later keys overwrite earlier ones
const base = { name: "Karma", age: 24 };
const updated = { ...base, age: 25 };
// { name: "Karma", age: 25 }
34 Optional Chaining ?.
Safety

Accessing keys that might not exist

Backend data isn't always complete. What if contact is missing? Normal access crashes the app.

optional-chaining.js
const trainee = { name: "Karma" };
// Note: no "contact" key!

// ❌ Without optional chaining — this crashes
console.log(trainee.contact.email);
// TypeError: Cannot read properties of undefined

// ✓ With optional chaining — safe
console.log(trainee.contact?.email);
// undefined (no error — just "nothing there")

// Combine with default value
const email = trainee.contact?.email ?? "not provided";

Rule: if a key might not exist, use ?. to be safe.

Wrap-up

Key takeaways

Objects group things
Related bits of data into one bundle. Access with .key or ["key"].
Arrays are ordered lists
map · filter · reduce · find · sort are the five you'll use every day.
JSON is just objects
Backend data arrives as objects. You already know how to read it.
Destructuring is a shortcut
Pull keys out cleanly. Use ?. when a key might not exist.
