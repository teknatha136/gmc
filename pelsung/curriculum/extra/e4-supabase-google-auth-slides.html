02 What you'll end up with

Working Google sign-in in your React app

  • A Sign in with Google button users can click
  • A session that survives page reloads (Supabase stores it for you)
  • Access to the user's email, name and avatar_url
  • A protected route that only signed-in users can see
  • A working sign out button

Same flow handles both sign in AND sign up — Google OAuth creates the user automatically the first time.

06
Part 1 · Supabase setup

Create your Supabase project

Steps 1–2 · About 5 minutes

07 Step 1 — Create a Supabase project
1
Go to supabase.com and click Start your project.
2
Sign in with GitHub (easiest).
3
Click New project. Pick any organization.
4
Give it a name (e.g. my-app) and a strong database password. Save the password — you may need it later.
5
Pick the region closest to you (e.g. Singapore for Bhutan).
6
Click Create new project. Wait ~2 minutes while it provisions.
08 Step 2 — Copy your project URL & anon key

In your Supabase dashboard, go to Project Settings → API. You need two values:

  • Project URL — looks like https://abcdefgh.supabase.co
  • anon public key — a long JWT starting with eyJ...
💡 Is the anon key secret?
No — it's safe to ship in your frontend. It only lets users do what your Row Level Security (RLS) rules allow. The service_role key (also on that page) IS secret — never put it in frontend code.

Part 2 · Google Cloud Console

Set up the OAuth client

Steps 3–7 · This is the fiddly part. Take your time.

10 Step 3 — Create a Google Cloud project
1
Go to console.cloud.google.com and sign in.
2
Click the project dropdown (top-left) → New Project.
3
Name it (e.g. my-app-auth). No org needed. Click Create.
4
Wait a few seconds, then make sure the new project is selected (top-left dropdown).
⚠️ Don't skip step 4
All your OAuth config goes inside the selected project. If you create the OAuth client in the wrong project, Google won't find it.
11 Step 4 — Configure the OAuth consent screen

Left sidebar → APIs & Services → OAuth consent screen.

1
Pick External (anyone with a Google account can sign in). Click Create.
2
App name: e.g. My App. User support email: your email.
3
Skip the App logo and domains for now (only needed for production).
4
Developer contact: your email. Click Save and continue.
5
Scopes — leave empty. Click Save and continue.
6
Test users — add your own Gmail address. Click Save.
12 Step 5 — Create the OAuth Client ID

Left sidebar → APIs & Services → Credentials.

1
Click + Create credentials → OAuth client ID.
2
Application type: Web application.
3
Name: e.g. my-app-web-client.
4
Now the redirect URIs — see the next slide.
13 Step 6 — Add the redirect URI (CRITICAL)

Under Authorized redirect URIs, add exactly one URL from your Supabase dashboard:

Authorized redirect URI
https://<YOUR-PROJECT-REF>.supabase.co/auth/v1/callback
⚠️ Where do you get this URL?
In Supabase dashboard → Authentication → Providers → Google, you'll see a "Callback URL (for OAuth)" field. Copy that exact URL and paste it into Google Cloud. Don't type it by hand — a single character off and Google rejects the login.

Click Create. Google shows you a Client ID and Client Secret. Keep that dialog open.

14 Step 7 — Copy your Google Client ID & Secret
  • Client ID — looks like 123456-abcdef.apps.googleusercontent.com
  • Client Secret — looks like GOCSPX-xxxxxxxxxxxxxxxx
🔒 Treat the Client Secret like a password
It goes into Supabase only — never commit it to git or paste it into your React code. If you ever leak it, regenerate it from Google Cloud Console.

Part 3 · Connect Google to Supabase

One field, then the magic happens

16 Step 8 — Enable Google provider in Supabase

Back in Supabase: Authentication → Providers → Google.

1
Toggle Enable Sign in with Google ON.
2
Paste your Client ID from Google.
3
Paste your Client Secret from Google.
4
Click Save. That's it — backend is fully wired.
✅ Quick sanity check
The "Callback URL (for OAuth)" shown on this Supabase page should match exactly the redirect URI you put in Google Cloud. If they don't match, fix it now.

Part 4 · React + TypeScript

Now the code

Steps 9–14 · You can copy each block straight into your project.

18 Step 9 — Install supabase-js

In your React project's terminal, run:

terminal
# Install the Supabase JS client npm install @supabase/supabase-js # That's it. No other auth library needed.

Supabase ships TypeScript types out of the box — no @types/... package needed.

19 Step 10 — Add environment variables

Create a .env.local file in your project root (same level as package.json):

.env.local
VITE_SUPABASE_URL=https://your-project-ref.supabase.co VITE_SUPABASE_ANON_KEY=eyJhbGciOi...your-long-anon-key
⚠️ Don't commit this file
Add .env.local to your .gitignore (Vite already ignores it by default). For Create React App projects, the prefix is REACT_APP_ instead of VITE_. Restart your dev server after creating this file.
20 Step 11 — Create the Supabase client
src/lib/supabase.ts
import { createClient } from '@supabase/supabase-js'; const supabaseUrl = import.meta.env.VITE_SUPABASE_URL; const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY; if (!supabaseUrl || !supabaseAnonKey) { throw new Error('Missing Supabase env vars — check .env.local'); } export const supabase = createClient(supabaseUrl, supabaseAnonKey);

One client, used everywhere. Import it wherever you need to talk to Supabase.

21 Step 12 — The Sign In with Google button
src/components/sign-in-button.tsx
import { supabase } from '../lib/supabase'; export function SignInButton() { const handleSignIn = async () => { const { error } = await supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: `${window.location.origin}/auth/callback`, }, }); if (error) console.error('Sign in failed:', error.message); }; return ( <button onClick={handleSignIn} className="flex items-center gap-3 rounded-lg border border-gray-300 bg-white px-5 py-2.5 text-sm font-semibold text-gray-700 shadow-sm hover:bg-gray-50" > <img src="https://www.svgrepo.com/show/475656/google-color.svg" alt="" className="h-5 w-5" /> Sign in with Google </button> ); }
22 Step 13 — Track who's signed in (useAuth hook)
src/hooks/use-auth.ts
import { useEffect, useState } from 'react'; import type { Session, User } from '@supabase/supabase-js'; import { supabase } from '../lib/supabase'; export type AuthState = { session: Session | null; user: User | null; loading: boolean; }; export function useAuth(): AuthState { const [session, setSession] = useState<Session | null>(null); const [loading, setLoading] = useState(true); useEffect(() => { // 1. read current session on mount supabase.auth.getSession().then(({ data }) => { setSession(data.session); setLoading(false); }); // 2. listen for sign in / sign out events const { data: { subscription } } = supabase.auth .onAuthStateChange((_event, newSession) => { setSession(newSession); }); return () => subscription.unsubscribe(); }, []); return { session, user: session?.user ?? null, loading }; }
23 Step 14 — Sign out
src/components/sign-out-button.tsx
import { supabase } from '../lib/supabase'; export function SignOutButton() { const handleSignOut = async () => { await supabase.auth.signOut(); }; return ( <button onClick={handleSignOut} className="rounded-lg bg-gray-900 px-4 py-2 text-sm font-semibold text-white hover:bg-gray-700" > Sign out </button> ); }

signOut() clears the session. Your onAuthStateChange listener will fire — the UI updates automatically.

24 Step 15 — Protected route
src/components/protected.tsx
import type { ReactNode } from 'react'; import { useAuth } from '../hooks/use-auth'; import { SignInButton } from './sign-in-button'; type Props = { children: ReactNode }; export function Protected({ children }: Props) { const { user, loading } = useAuth(); if (loading) { return <div className="p-8 text-gray-500">Loading...</div>; } if (!user) { return ( <div className="flex min-h-screen items-center justify-center"> <SignInButton /> </div> ); } return <>{children}</>; }

29 Common pitfalls (read these before you panic)
redirect_uri_mismatch
The URL in Google Cloud doesn't exactly match Supabase's callback URL. Copy-paste — don't retype.
"App not verified" warning
Normal in development. Click Advanced → Go to [app] (unsafe). Submit to Google for verification before going public.
Env vars come back undefined
Restart npm run dev after editing .env.local. Vite only reads env vars at startup.
"Access blocked: This app's request is invalid"
You forgot to enable the Google provider in Supabase, or the Client Secret is wrong. Re-check Step 8.
User redirects to a blank page
Your redirectTo URL doesn't have a route. Either remove it (Supabase defaults to your site URL) or add a route at /auth/callback.
Session lost on refresh
Supabase auto-persists to localStorage. If lost, you may have created two clients — keep only one in src/lib/supabase.ts.
Only test users can sign in
Your OAuth consent screen is in "Testing" mode. Add the user as a Test User, or "Publish" the app for general access.
CORS errors in console
In Supabase → Authentication → URL Configuration, add your dev URL (e.g. http://localhost:5173) under Site URL or Redirect URLs.
30 Before you ship to production

Production checklist

  • Add your production URL to Google Cloud's Authorized redirect URIs (the Supabase callback stays the same — it's the same Supabase project)
  • Add your production URL to Supabase's Site URL and Redirect URLs
  • Set the same env vars (VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY) in your hosting provider (Vercel, Netlify, etc.)
  • Submit your app to Google for OAuth verification (removes the scary warning screen)
  • Set up Row Level Security (RLS) on every Supabase table — never ship without it
  • Add a custom logo and privacy policy URL in the OAuth consent screen
