02 Why bother?
🤔 Honest question

"I can already write text in Word.
Why do I need another way to write?"

Because Word files don't work in GitHub, in bug trackers, in AI chats, in team wikis, or in code.
Markdown does. Every tool you will use in software speaks it.

03 Definition
Extra · E2

What is Markdown?

Plain text with tiny marks that turn into formatted text

You write this
# My Report

I visited **Gelephu** last week.

Three things I noticed:
- Fast internet
- New buildings
- [Friendly people](https://gmc.bt)
People see this

My Report

I visited Gelephu last week.

Three things I noticed:

  • Fast internet
  • New buildings
  • Friendly people

That's it. The whole language fits on one page. You can learn it today.

Part 1 · The Syntax

The whole
language

10 things. That's it. Let's walk through each.

01
06 1/10 · Headings

Headings · use #

One # = biggest. Add more # = smaller.

Write
# Project Plan
## Phase 1
### Week 1 goals
#### Daily stand-ups
Shown

Project Plan

Phase 1

Week 1 goals

Daily stand-ups

⚠️ Space after the #. #Hello does nothing. # Hello is a heading.

07 2/10 · Bold & italic

Emphasis

Stars wrap around the word you want to highlight

Italic *tired* tired Bold **important** important Bold italic ***urgent*** urgent Strikethrough ~~wrong~~ wrong
Trick to remember: one star = mild (italic), two stars = strong (bold). Same as emphasis in speech.
08 3/10 · Lists

Lists

Use - for bullets or 1. for numbered. Indent to nest.

Write
Groceries:
- Rice
- Chili
  - Red
  - Green
- Oil

Steps:
1. Open laptop
2. Open browser
3. Go to gmail.com
Shown

Groceries:

  • Rice
  • Chili
    • Red
    • Green
  • Oil

Steps:

  1. Open laptop
  2. Open browser
  3. Go to gmail.com

💡 Numbered lists auto-renumber — if you write 1. 1. 1. it still shows 1. 2. 3.

09 4/10 · Links & images

Links & images

Square brackets for the text, parens for the URL.

Write
Visit [SELISE](https://selisegroup.com)
for more info.

Logo:
![SELISE logo](logo.png)

Bare URL:
<https://selisegroup.com>
Shown

Visit SELISE for more info.

Logo:
📷 SELISE logo

Bare URL:
https://selisegroup.com

Pattern: [text](url) is a link. Add ! in front — ![alt](url) — and it's an image.

10 5/10 · Code

Code · backticks

One backtick for a word. Three backticks for a whole block.

Write
Run the `npm install` command.

```js
function greet(name) {
  return "Hello, " + name;
}
```
Shown

Run the npm install command.

function greet(name) {
  return "Hello, " + name;
}

Add the language after the first three backticks (```js, ```python) — GitHub and most tools will color the code.

11 6/10 · Quotes & dividers

Quotes · horizontal rules

Write
> "The best way to predict
> the future is to invent it."
> — Alan Kay

---

Next topic below the line.
Shown
"The best way to predict the future is to invent it."
— Alan Kay

Next topic below the line.

> at the start of a line makes it a quote. --- on its own line draws a divider.

12 7/10 · Tables

Tables · pipes & dashes

Column dividers are |. The second row (with ---) separates header from body.

Write
| Name    | Vertical | Status |
|---------|----------|--------|
| Karma   | Dev      | Active |
| Pema    | QA       | Active |
| Tashi   | BA       | Review |
Shown
NameVerticalStatus
KarmaDevActive
PemaQAActive
TashiBAReview

💡 Don't hand-align spaces — most editors have "Format Table" which does it for you. Or just leave it messy; it still works.

13 8/10 · GitHub flavor

GitHub-Flavored Markdown

GitHub added a few extras you'll use all the time.

Write
Today's tasks:
- [x] Write spec
- [ ] Review with Tek
- [ ] Share with team

Tag people: @bhuwan
Link an issue: #42
Emoji: :rocket: :bug: :tada:
Shown

Today's tasks:

  • Write spec
  • Review with Tek
  • Share with team

Tag people: @bhuwan
Link an issue: #42
Emoji: 🚀 🐛 🎉

These only work on GitHub & tools that copy GitHub's flavor (most modern ones do).

14 9/10 · Escaping

"But what if I want to write a star?"

Put a backslash \ before any Markdown character to show it literally.

Write
This becomes italic: *hello*

This stays as stars: \*hello\*

Show a hash: \# not a heading

Show a backslash: \\
Shown

This becomes italic: hello

This stays as stars: *hello*

Show a hash: # not a heading

Show a backslash: \

You rarely need this — but remember it exists for that one moment when Markdown is eating your text.

15 10/10 · The weird one

Line breaks · the gotcha

Markdown ignores single line breaks. Use a blank line between paragraphs.

Write
Line one.
Line two.

Line three after a blank line.
Shown

Line one. Line two.

Line three after a blank line.

The #1 thing beginners get wrong. If your text isn't breaking into new lines, you're missing a blank line between the paragraphs.

20 Markdown + AI
Bonus · for every vertical

Markdown is the AI language

When you write prompts to Claude Code, ChatGPT, or any AI — use Markdown. Structure the request. Number your asks. The AI understands it better than a wall of text.

# Task
Build a small React counter component.

# Requirements
- Start at 0
- Buttons: +1, -1, reset
- Use Tailwind for styles

# Constraint
Keep it in one file.

A structured prompt = a structured answer. A messy prompt = a messy answer.

23 Exercise 1 of 3
15 minutes

Exercise 1 · Write your "About Me"

The task

Write a short personal profile in Markdown. Open the playground and type it. No help, no copy-paste.

It must include:

  • A big heading with your name
  • A smaller heading "About Me" and a paragraph about yourself
  • A list of 3 things you like (use bullets)
  • A list of 3 goals for Pelsung (use numbers)
  • A link to a website you visit daily
  • Something in italic and something in bold
  • A blockquote with a quote you like

✅ Done when the right-hand preview looks like a real profile page.

24 Exercise 2 of 3
15 minutes

Exercise 2 · File a fake bug

The task

Pretend you found a bug in Facebook. Write the bug report in Markdown, in the format a developer would actually read.

Must include:

  • Title (H1) — one clear sentence, not "bug bug bug"
  • Environment — device, browser, OS (use bold labels)
  • Steps to reproduce — a numbered list of 3+ steps
  • Expected vs Actual — two sub-headings (H2)
  • A code snippet somewhere — use backticks (e.g. the URL or an error)
  • A task list (checkboxes) of things still to verify

💡 Tip: copy the QA example from slide 18 and change every field. Don't leave anything the same.

25 Exercise 3 of 3
15 minutes

Exercise 3 · A comparison table

The task

Build a Markdown table comparing 3 apps you use every day (WhatsApp, Instagram, Druk Pay, BoB, Pelrithang — your choice).

Columns:

  • App name
  • What you use it for
  • One thing you like
  • One thing that annoys you

This is the kind of document a BA would write in a discovery meeting.

27 Gotchas

The 5 things beginners always get wrong

❌ Missing space after #

#Hello is broken. # Hello works.

❌ Forgot blank lines

Two lines with no blank between = one paragraph. Always put a blank line between blocks.

❌ Wrong backticks

Curly quotes (', ") don't work. Use straight backtick (`) — below Esc key.

❌ Tables without the --- row

No dash row = no table. Just ugly pipes and text.

❌ Nested lists with 1 space

Indent 2 spaces (or tab) to nest. One space won't nest reliably.

✅ When in doubt — preview

Always look at the rendered side before you send it. Saves embarrassment.
