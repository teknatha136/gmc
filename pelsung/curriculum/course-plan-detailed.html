Dev 06 Vibe Coding with Claude Code Claude Code CLI, CLAUDE.md, Prompt engineering

Topics

  • What is "vibe coding" — the modern dev mindset shift: describe what you want, AI writes it
  • Claude Code CLI: installation, authentication, basic usage
  • CLAUDE.md files: how they guide Claude's behavior per project (conventions, stack, rules)
  • Prompt engineering for code: being specific, providing context, iterating on output
  • Reading and reviewing AI-generated code — never blindly accept

Trainer Notes

  • Pre-setup required: Each trainee's machine should have Claude Code installed and authenticated before class. Pre-configure a project with an opinionated CLAUDE.md that enforces the stack used in this program (React, Node/Express, etc.), coding conventions, and guardrails (e.g., "always use TypeScript", "write tests for new functions"). This way, even beginners produce consistent, well-structured code from day one.
  • Equalizer class. Claude Code partially closes the gap between beginners and advanced. Beginners can describe what they want in plain English and get working code. But they MUST understand what the code does — don't let them skip comprehension.
  • Yugel Tshering is already training in GenAI/prompt engineering — let him share his experience with prompting techniques.
  • Important warning for advanced students: AI can make you lazy. Emphasize that understanding > generating. They should use Claude Code to accelerate, not replace thinking. Show them how to use /diff to review changes before accepting.
  • For beginners: live demo — type a plain English request into Claude Code ("create a webpage with a nav bar and 3 cards showing tourist spots in Gelephu"), watch it generate code, then walk through every line together. Build confidence that they can "direct" code even if they can't write it from scratch yet.
  • Show the CLAUDE.md file explicitly. Explain: "This file tells Claude what kind of project this is, what rules to follow, what tools to use. When you change this file, Claude changes its behavior." This teaches them that AI is configurable, not magic.
Dev 07 AI-Assisted Development Workflows Claude Code workflows, Pre-configured projects, AI literacy

Topics

  • Claude Code workflows: scaffolding a project, adding features, fixing bugs, refactoring
  • Working with pre-configured CLAUDE.md: how the guardrails shape output (stack, style, testing)
  • Multi-step tasks: breaking a feature request into prompts (plan → implement → test → review)
  • AI literacy: reviewing generated code, identifying errors, understanding file structure
  • When to use AI vs. writing from scratch — knowing the difference
  • Git workflow with Claude Code: committing AI-generated code, meaningful commit messages

Trainer Notes

  • Hands-on session. Each trainee uses Claude Code with the pre-configured project to build a small feature (e.g., "add a contact form that saves to the database"). Advanced students go further: "add form validation, error handling, and a success toast notification."
  • Sonam Choki and Sonam Rinchen can help explain the generated code structure to nearby beginners — they understand project organization and can spot when Claude makes bad decisions.
  • Tashi Phuntsho (3yr IT mgmt) should review Claude's output critically — show the class that even strong developers use AI, but they READ every line. Demo his review process.
  • Key lesson: The CLAUDE.md file is your "senior developer" that's always watching. Show how changing one line in CLAUDE.md (e.g., adding "always include error handling") changes Claude's output for the entire project. This is powerful and practical.
  • Exercise: Everyone generates the same feature with Claude Code but with different CLAUDE.md configurations. Compare outputs. Discuss which configuration produced better code and why. This teaches that the human's job is to configure and direct, not just accept.
